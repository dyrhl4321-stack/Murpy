## 0. 절대 규칙 (오늘 데인 것들 — 꼭 지키기)

1. **근본(공통 자산)을 함부로 바꾸지 말 것.**
   - 문제가 "후드 입었을 때"면 **후드만** 고친다. 기본 캐릭터(walk.png 티셔츠 색 등)를 임의로 바꾸지 말 것.
   - base를 바꾸면 모든 것에 영향 → 되돌리기 대참사. **국소적으로 해결.**
2. **로컬 이미지 바꾸면 반드시 "캐시 버전"을 올릴 것 (제일 많이 데인 것).**
   - 서비스워커(sw.js)가 로컬 이미지를 **캐시 우선**으로 저장 → 파일만 바꾸면 **화면에 안 바뀜**(Ctrl+Shift+R로도 안 뚫림).
   - **아이템(char/items/*)** 바꿨으면 → `index.html`의 `window._ITEM_V` 숫자 +1 (URL에 `?v=` 붙어 캐시 우회).
   - **walk.png / heltori.png 등** 바꿨으면 → `index.html`·`char/_layer_preview.html`의 `walk.png?v=N` **현재 번호를 grep으로 확인하고** +1. (추측으로 sed 하지 말 것! 오늘 v10인데 v8→v9로 바꿔서 안 먹혔음)
   - 배포 땐 `sw.js`의 `-v79`류 버전도 +1.
   - 확인: `grep -on "walk\.png?v=[0-9]*" index.html | sort -u`, `grep "_ITEM_V" index.html`
3. **"고쳤다"고 말하기 전에 항상 렌더해서 눈으로 확인.**
   - 파일이 맞아도 캐시/버전 때문에 대표 화면엔 옛것일 수 있음. 내가 PIL로 렌더해서 확인 후 보고.

## 1. 아이템 추가 워크플로우 (역할 분담)

**대표(아트):**
- ① **아이템 아이콘** PNG (썸네일용, 투명배경) → `캐릭터스프라이트시트/아이템/` 폴더
- ② **그 아이템을 우리 캐릭터에 씌운 4방향 스프라이트 시트** PNG (추출용, 투명배경)
- 프롬프트 팁: *"이 도트 스프라이트 시트의 구조와 톤앤매너를 유지한 채 ○○ 착용한 스프라이트 시트. 옷·바지·신발 등 다른 것 절대 건들지 말기. 배경 투명 PNG(JPEG 금지)."*
- **색이 팔레트에서 유일할수록** 추출이 깨끗함(레드 후디 = 최고). 기본 파란 티셔츠와 겹치는 파랑은 까다로움.

**나(코드):**
- `char/extract_item.py`로 **아이템만 추출** → 레이어 시트(`char/items/<id>.png`) 만들고
- **썸네일은 대표가 준 아이콘**을 씀(캐릭터에서 자동크롭 ❌). 엄격 알파(>50)로 여백 바짝 크롭.
- `index.html` `CHAR_ITEMS`에 `{id, slot, name, sheet, thumb}` 등록 + `_ITEM_V` +1.

## 2. 규격

- **스프라이트 시트**: 3열(정지·걸음A·걸음B) × 4행(아래·위·좌·우) = 12프레임, 투명 PNG. 4방향 각각 그려 **미러 없음**.
- **우리 격자 셀**: 사람 **282×448** (walk.png, 2026-07-07 2배 고화질 전환). 정규화: 캐릭터 높이 428, 바닥 정렬, 가로 중앙.
  - 대표 소스는 고해상도(예: 착용시트 704×1492)로 주면 됨 — 2배 추출은 `extract_item.py`가 자동. 작업방식 변경 없음.
- **슬롯 z순서(뒤→앞)**: `body → bottom → shoes → top → hair → hat → acc`
  - `acc`(헤드셋 등)는 **최상단**(모자 위). `mask` 슬롯 코드는 남아있으나 UI는 "악세사리(acc)" 탭으로 대체됨.

## 3. extract_item.py 사용법

`char/CONFIG` 딕셔너리만 바꿔 실행하거나, 다른 스크립트에서 `E.build(cfg, judge=색판정함수)` 호출.

**핵심 옵션:**
- `src` / `out` / `rows` / `cols`: 소스 파일·출력이름·격자밴드(알파 투영으로 측정, `detect_grid()`)
- `region=(y0,y1)`: 아이템 있는 세로비율
- `mode`: `"region"`(영역내 색추출, 기본) / `"flood"`(상단에서 연결된 것만 → **모자류 밑단 아치·눈도트 방지**)
- `per_frame=True`: **상의류**(팔 움직임 → 프레임별 추출). 모자·악세는 안 씀(idle 복제)
- `min_component=N`: 작은 잔여도트(눈 하이라이트 등) 제거
- `dilate=N`, `dilate_ymin`, `dilate_solid`: 팽창(채도 높은 이웃 우선 → 회색외곽선 안 퍼짐). 지금은 거의 cover_tee로 대체
- `cover_tee=True`: **상의류 필수**. 기본 파란 티셔츠 삐짐을 가장 가까운 상의색으로 번져 메꿈(음영 이어짐, base 안 건드림)

**judge(색 판정) 예:**
- 검정 모자: 중성 어두움`(br<98, br-mn<24)` + 흰 브림`(mn>148)`. 갈색머리는 제외됨.
- 은색 헤드폰: `mn>112, max-min<38`. + `bridge_vert`(밴드↔이어컵 머리에 가려 끊긴 곳 세로 연결)
- 레드 후디: `r>70, r>g+25, r>b+22, g<130` (빨강)
- 네이비 져지: 파랑 + 흰숫자 + 빨간테두리

**아이템 타입별 정석:**
| 타입 | mode | per_frame | 추가 |
|---|---|---|---|
| 모자(캡·비니) | flood | X | 밑단 아치 자연스러움 |
| 악세(헤드셋) | region | X | min_component + bridge_vert |
| 상의(후디·져지) | region | O | **cover_tee=True** 필수 |

## 5. 배포 체크리스트

1. 바뀐 로컬 이미지에 맞춰 **버전 올렸나?** (`_ITEM_V`, `walk.png?v=`, `sw.js`)
2. 인라인 스크립트 문법검사: `node --check`(비-module 블록 추출)
3. `MW_DEV_ALWAYS`(index.html) — 개발용 매번 인트로. **진짜 출시(뿌리기) 때 false로.**
4. main 브랜치 = GitHub Pages 자동 배포. push 전 로컬 육안 확인.
