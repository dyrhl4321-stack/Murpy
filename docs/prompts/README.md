# 나노바나나(제미나이) 프롬프트 라이브러리

머피월드 도트 에셋을 나노바나나(제미나이)로 뽑을 때 **재활용하는 프롬프트 모음**.
새 에셋을 만들기 전에 해당 카테고리 문서를 열어 **[공통 규칙] + [카테고리 프롬프트]** 를 조합해 쓴다.

> 왜 문서로? 매번 프롬프트를 새로 쓰면 규칙(초록배경·미러금지·워터마크 등)을 빠뜨려 다시 뽑게 된다.
> 여기 한곳에 요약해두고 복붙하면 실패가 준다.

## 모든 프롬프트 공통 규칙 (나노바나나 하드룰)

1. **배경 = 단색 형광초록 `#00FF00`.** 투명 배경은 못 그린다 → 초록으로 뽑고 누끼(Photoroom).
2. **워터마크·반짝이(✦)·불필요한 글자·그림자 금지.** 그래도 ✦는 자주 찍힌다 → 아래 제거 툴로 뗀다.
3. **픽셀 아트, 선명한 계단식 도트. 안티에일리어싱·블러 금지.**
4. **입체 소품 금지** — 가방·손에 든 물통처럼 앞/뒤/옆이 확 다른 물건은 프레임마다 어긋난다.
   정체성은 **납작한 마커**로: 옷 프린트(로고·문구), 머리·모자·헤어밴드, 표정. (헬토리도 백팩 빼고 셔츠 프린트로 감.)
5. **레퍼런스로 '빡빡이(기본 체형)'를 쓰면** 민머리·헐벗음을 따라간다 → "머리·옷 반드시 입혀라" 오버라이드를 넣는다(카테고리 문서에 포함).

## 워터마크(제미나이 ✦) 제거

- **툴:** `char/remove_gemini_watermark.py` — **도너 프레임 이식 방식**(없는 픽셀을 지어내지 않고,
  같은 시트의 다른 프레임에서 그 자리를 정합해 마스크 안쪽만 통째로 옮겨 심는다).
  고전 인페인트(OpenCV Telea)는 별 모양 얼룩이 남아 쓰지 않는다.
- **입력:** 누끼 **전** 원본, 크기 **1408×3008**(3열×4행). ✦는 항상 **우하단(오른쪽·걷기B)** 프레임에 찍힘.
- **사용:**
  ```
  python char/remove_gemini_watermark.py "원본.png" -o "출력폴더"
  ```
  - 여러 장 한 번에: 파일 경로를 여러 개 나열.
  - **`--force`**: 흐린 배경(흰옷) 위라 워터마크가 임계값 미달로 안 잡히거나, 머리 흔들림으로
    도너가 안 맞아 "중단"되면 붙인다. 판정을 건너뛰고 최적 도너를 무조건 이식 → **반드시 눈으로 검증.**
- **출력:** `<원본>_clean.png` (초록 배경 유지). 이후 Photoroom 누끼 → `_clean-Photoroom.png`.
- **의존성:** `numpy`, `pillow` (이 머신은 `python -m pip install numpy` 필요했음).

## 후처리 흐름 (에셋 파이프라인)
```
원본(초록 #00FF00 배경)
  └─▶ [워터마크 제거] char/remove_gemini_watermark.py  →  _clean.png (초록 배경)
        └─▶ [누끼] Photoroom                            →  _clean-Photoroom.png (투명)
              └─▶ [앱 추출] char/build_walk.py 등        →  게임용 시트(char/*.png)
```
