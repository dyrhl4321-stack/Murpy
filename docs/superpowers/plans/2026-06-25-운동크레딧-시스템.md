# 운동 크레딧 시스템 (Proof of Workout) 구현 계획

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 운동 인증으로 크레딧을 쌓고, 매칭·대숲을 여는 선순환을 만든다. 지금은 베타(표시+무제한+적립), 추후 한 줄로 정식(차감·게이트) 전환.

**Architecture:** 단일 `index.html`에 크레딧 코어(모듈 스코프 함수 묶음)를 추가하고, 인증 글 작성 성공/로그인/매칭·대숲 작성 지점에 훅을 건다. `CREDITS_MODE` 상수로 beta/enforced 전환. 잔액은 `users/{uid}` 문서의 `credits`·`creditsLastEarned` 필드에 저장.

**Tech Stack:** 순수 HTML/JS 단일 파일, Firebase 10.12.0 ES module(Firestore), GitHub Pages. 테스트 프레임워크 없음 → 브라우저 수동 검증.

## Global Constraints

- 이모지 금지 → 머피 라인 SVG만. 크레딧은 골드(`var(--gold)` = #F5C24B). 블루 #3D7EFF.
- 경제 상수(조정 가능): `CREDIT_WELCOME = 10`, `CREDIT_DAILY = 1`, `COST_MATCH = 1`, `COST_BAMBOO = 1`.
- `CREDITS_MODE` 기본값 `'beta'`. `'beta'`=표시+적립+무제한(차감/차단 없음), `'enforced'`=실제 차감·게이트.
- 베타 중엔 버튼별 "N 크레딧" 라벨을 달지 않는다(무제한이라 혼란). 시스템 존재 인지는 크레딧 칩 + "베타·무제한" 배지로, 비용 안내는 정식 모드에서 잔액 부족 시 토스트로만.
- Firebase 모듈 스코프: 크레딧 함수는 `<script type="module">` 안에 두고 `window.*`로 등록. 일반 onclick에서 호출 가능.
- ⚠️ enforced 전환 전 잔액 무결성(보안규칙/Functions)은 별도 선행 과제(이 계획 범위 밖, 베타는 무제한이라 위험 없음).
- 각 태스크 커밋 시 `index.html` cache-bust + `sw.js` 버전(`murpy-vN`) **둘 다** 갱신 후 `git push`(자동 새로고침 동작 조건).
- 테스트는 수동 시나리오.

```javascript
// ===== 운동 크레딧 (Proof of Workout) =====
window.CREDITS_MODE = 'beta'; // 'beta'(표시+적립+무제한) | 'enforced'(차감·게이트). 사용자 충분히 모이면 'enforced'.
const CREDIT_WELCOME = 10, CREDIT_DAILY = 1, COST_MATCH = 1, COST_BAMBOO = 1;
window._creditState = { credits: 0, lastEarned: '', ready: false };

function _creditToday() {
  const d = new Date();
  return d.getFullYear() + '-' + String(d.getMonth() + 1).padStart(2, '0') + '-' + String(d.getDate()).padStart(2, '0');
}

// 로그인 시 1회: 잔액 없으면 환영 보너스로 초기화, 있으면 캐시
window.ensureCreditsInit = async (uid) => {
  if (!uid) return;
  try {
    const ref = doc(db, 'users', uid);
    const snap = await getDoc(ref);
    const d = snap.exists() ? snap.data() : {};
    if (typeof d.credits !== 'number') {
      await setDoc(ref, { credits: CREDIT_WELCOME, creditsLastEarned: '' }, { merge: true });
      window._creditState.credits = CREDIT_WELCOME;
      window._creditState.lastEarned = '';
    } else {
      window._creditState.credits = d.credits;
      window._creditState.lastEarned = d.creditsLastEarned || '';
    }
    window._creditState.ready = true;
    if (window.renderCreditUI) window.renderCreditUI();
  } catch (e) {}
};

// 하루 첫 운동 인증 시 +CREDIT_DAILY (이미 오늘 받았으면 무시)
window.grantDailyCreditOnVerify = async (uid) => {
  if (!uid) return;
  const today = _creditToday();
  if (window._creditState.lastEarned === today) return;
  try {
    await updateDoc(doc(db, 'users', uid), { credits: increment(CREDIT_DAILY), creditsLastEarned: today });
    window._creditState.credits += CREDIT_DAILY;
    window._creditState.lastEarned = today;
    if (window.renderCreditUI) window.renderCreditUI();
    if (window.showToast) showToast('+' + CREDIT_DAILY + ' 크레딧 · 오늘 운동 인증 완료!');
  } catch (e) {}
};

window.getCredits = () => window._creditState.credits;

// 소모: beta는 항상 통과(무제한), enforced는 잔액 차감/부족 시 false
window.spendCredit = async (uid, cost) => {
  if (window.CREDITS_MODE !== 'enforced') return true;
  if (!uid || window._creditState.credits < cost) return false;
  try {
    await updateDoc(doc(db, 'users', uid), { credits: increment(-cost) });
    window._creditState.credits -= cost;
    if (window.renderCreditUI) window.renderCreditUI();
    return true;
  } catch (e) { return false; }
};

```
