# 머피 적립 "m-토큰" 축하 연출 + 화폐명 통일 — 구현 계획

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (권장) 또는 superpowers:executing-plans 로 task 단위 구현. 단계는 체크박스(`- [ ]`)로 추적.

**Goal:** 피드(운동 인증) 작성으로 머피가 적립되는 순간을 중앙 "m-토큰" 축하로 연출하고, 화면 문구·아이콘을 "머피"/m-토큰으로 통일한다.

**Architecture:** 단일 `index.html`에 (1) 재사용 m-토큰 SVG 헬퍼, (2) 중앙 축하 오버레이(DOM+CSS+JS), (3) 첫 적립 안내 카드를 추가하고, 피드 제출부에서 기존 토스트/하단 시트 호출을 새 연출로 교체한다. 적립 로직·Firestore 필드는 불변, 화면 표현만 변경.

**Tech Stack:** 바닐라 JS (ES module + `window` 브릿지), CSS keyframes, 기존 색상 시스템(보라 #A78BFA / 골드 #F5C24B). 빌드 없음, GitHub Pages 배포.

## Global Constraints

- 대상 파일은 `index.html` 단일 파일 + `sw.js`(캐시 버전). 새 외부 의존성 금지.
- **이모지 미사용** — 머피 라인 SVG만.
- 내부 식별자 불변: `credits`/`creditsLastEarned` Firestore 필드, `CREDITS_MODE`, `CREDIT_DAILY`, 상수·함수·DOM id(`#credit-amount` 등). **사용자에게 보이는 텍스트만** "크레딧"→"머피".
- 단위 표기는 붙여 씀: **"N머피"** (예: "1머피", "이제 12머피").
- 색상: 머피=보라 #A78BFA, 강조 스파클=골드 #F5C24B, CTA=블루 #3D7EFF.

- `prefers-reduced-motion: reduce` 존중.
- **테스트 프레임워크 없음.** 각 task의 검증 게이트 = (A) 인라인 스크립트 문법검사(아래 명령) + (B) 로컬 브라우저 수동 확인.
