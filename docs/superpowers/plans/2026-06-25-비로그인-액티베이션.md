# 비로그인 액티베이션 (소비형 아하 모먼트) 구현 계획

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 비로그인 방문자를 소비형 경험(실시간 활기 + 고의도 JIT 로그인 전환)으로 빠르게 아하 모먼트에 연결해 가입 전환을 높인다.

**Architecture:** 단일 `index.html`의 기존 자산 두 개(`renderHomeLiveBar`, `requireLogin`+`login-guard-modal`)를 다듬는다. 라이브바는 콜드스타트에서 가짜 숫자 없이 정직한 3단 폴백으로 항상 살아있어 보이게 하고, 고의도 액션은 맥락형 로그인 시트로 전환하며 "운동 인증하기"에 한해 로그인 후 의도(사진 선택기)를 재개한다.

**Tech Stack:** 순수 HTML/CSS/JS 단일 파일, Firebase 10.12.0 ES module(Firestore/Auth), GitHub Pages 배포. 테스트 프레임워크 없음 → 검증은 브라우저 수동 시나리오.

## Global Constraints

- 이모지 금지 → 머피 전용 라인 SVG 아이콘만 사용.
- 색상: 블랙/화이트 + 블루 `#3D7EFF`(액션) / 골드 `#F5C24B`(별점·뱃지).
- 콜드스타트 정직성: 가짜 카운트·더미 데이터 금지. 숫자는 실제 집계만.
- `LIVE_MIN = 3` (이 수 미만이면 다음 범위로 폴백).
- 비로그인 익명 게시 금지 → Firestore 보안규칙·작성자 귀속 불변.
- Firebase 모듈 스코프 주의: `onclick`에서 쓰는 함수는 일반 `<script>`에 두거나 `window.*` 등록. 기존 패턴 따를 것.
- 각 태스크 커밋 시 상단 `<!-- cache-bust: vYYYYMMDDNNN -->` 갱신 후 `git push`(= GH Pages 배포).
- 테스트는 자동화 불가 → 각 태스크의 "검증"은 명시된 브라우저 수동 시나리오로 수행.

- [ ] **Step 2: `renderHomeLiveBar` 블록을 아래로 교체**

기존 `// ===== 오늘 운동 인증 라이브 바 ...` 주석부터 함수 닫는 `}`까지(약 4689–4720) 전체를 다음으로 교체:

```javascript
// ===== 오늘 운동 인증 라이브 바 (고유 유저 집계, 콜드스타트 3단 폴백) =====
const LIVE_MIN = 3; // 이 수 미만이면 다음 범위로 폴백 (가짜 숫자 금지)

function _liveBarAgg(snap) {
  const seen = new Set();
  const recentPhotos = [];
  snap.docs.forEach(d => {
    const p = d.data();
    if (!p.userId || seen.has(p.userId)) return;
    seen.add(p.userId);
    if (recentPhotos.length < 3) {
      recentPhotos.push(window._cdnImg(userPhotoCache.get(p.userId) || p.userPhoto || '', 60));
    }
  });
  return { n: seen.size, recentPhotos };
}

function _liveBarHTML(recentPhotos, label) {
  const avatars = recentPhotos.map((ph, i) => ph
    ? `<img src="${ph}" style="width:24px;height:24px;border-radius:50%;border:2px solid #0a0a0a;object-fit:cover;${i ? 'margin-left:-9px' : ''}">`
    : `<div style="width:24px;height:24px;border-radius:50%;border:2px solid #0a0a0a;background:linear-gradient(135deg,#2563EB,#7C3AED);${i ? 'margin-left:-9px' : ''}"></div>`).join('');
  const flame = `<svg width="13" height="13" viewBox="0 0 24 24" fill="none" stroke="#FF7A45" stroke-width="2" stroke-linecap="round" stroke-linejoin="round" style="vertical-align:-2px;margin-left:4px"><path d="M8.5 14.5A2.5 2.5 0 0 0 11 12c0-1.38-.5-2-1-3-1.072-2.143-.224-4.054 2-6 .5 2.5 2 4.9 4 6.5 2 1.6 3 3.5 3 5.5a7 7 0 1 1-14 0c0-1.153.433-2.294 1-3a2.5 2.5 0 0 0 2.5 2.5z"/></svg>`;
  return `
    <div style="padding:9px 14px;background:rgba(255,255,255,0.04);border:1px solid rgba(255,255,255,0.08);border-radius:12px;display:flex;align-items:center;gap:9px">
      <div style="display:flex;align-items:center;flex-shrink:0">${avatars}</div>
      <span style="font-size:12px;color:rgba(255,255,255,0.7);font-weight:600">${label}${flame}</span>
    </div>`;
}

async function renderHomeLiveBar() {
  const bar = document.getElementById('home-live-bar');
  if (!bar) return;
  try {
    const today = new Date(); today.setHours(0, 0, 0, 0);
    // Tier 1: 오늘
    const todaySnap = await getDocs(query(collection(db, 'feed'), where('createdAt', '>=', today), orderBy('createdAt', 'desc')));
    const t = _liveBarAgg(todaySnap);
    if (t.n >= LIVE_MIN) {
      bar.innerHTML = _liveBarHTML(t.recentPhotos, `오늘 <span style="color:#fff;font-weight:800;font-size:13px">${t.n}명</span>이 운동 인증했어요`);
      bar.style.display = 'block';
      return;
    }
    // Tier 2: 최근 7일
    const weekStart = new Date(today); weekStart.setDate(weekStart.getDate() - 6);
    const weekSnap = await getDocs(query(collection(db, 'feed'), where('createdAt', '>=', weekStart), orderBy('createdAt', 'desc')));
    const w = _liveBarAgg(weekSnap);
    if (w.n >= LIVE_MIN) {
      bar.innerHTML = _liveBarHTML(w.recentPhotos, `이번 주 <span style="color:#fff;font-weight:800;font-size:13px">${w.n}명</span>이 운동 인증 중`);
      bar.style.display = 'block';
      return;
    }
    // Tier 3: 정성적 (숫자 없음)
    if (w.n >= 1) {
      bar.innerHTML = _liveBarHTML(w.recentPhotos, `운동 인증이 모이고 있어요`);
      bar.style.display = 'block';
      return;
    }
    // Tier 4: 전무 → 숨김
    bar.style.display = 'none';
  } catch (e) { bar.style.display = 'none'; }
}
```

- [ ] **Step 1: `requireLogin` 교체 + `dismissLoginGuard` 추가**

기존(약 2148):
```javascript
function requireLogin(reason) {
  document.getElementById('login-guard-text').textContent = reason || '이 기능은 로그인 후 사용할 수 있어요';
  openModal('login-guard-modal');
```

```javascript
function requireLogin(reason, onAfterLogin) {
  document.getElementById('login-guard-text').textContent = reason || '이 기능은 로그인 후 사용할 수 있어요';
  window._pendingIntent = (typeof onAfterLogin === 'function') ? onAfterLogin : null;
  openModal('login-guard-modal');
}
// 로그인 없이 가드 모달을 닫을 때는 보관된 의도를 폐기 (엉뚱한 시점 실행 방지)
function dismissLoginGuard() {
  window._pendingIntent = null;
  closeModal('login-guard-modal');
}
```
