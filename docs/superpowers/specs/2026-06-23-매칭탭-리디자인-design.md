# 매칭 탭 리디자인 + Firestore 실연동 — 설계

작성일: 2026-06-23

## 배경 / 문제

- 매칭 탭 카드가 더미데이터(`PROFILES`)로 동작 → 카드 탭 시 실제 운동 기록을 못 보여줌
- 스와이프 제스처가 오히려 불편 (사용자 피드백)
- "오늘의 추천 N/M" 진행바가 초기 사용자 부족(희소성)을 광고함
- 매칭 전/후 차이가 약함: 매칭 전에 이미 사진·센터·경력·소개 다 보이고, 매칭 후엔 피드 모자이크 해제 + 채팅뿐 → "매칭하면 뭐가 좋아지지?"가 약함

## 콜드스타트 원칙

- 실명·실제 헬스장 기반 커뮤니티라 **더미/가짜 프로필 0** (들통나면 신뢰 붕괴)
- 시드 커뮤니티 = GBD CREW 247명. 범위를 좁혀 밀도를 만든다
- **희소성(카운트) 숨김** — 진행바/카운트 노출 제거
- 크루 기능은 손대지 않음 (이미 GBD CREW 담을 만큼은 됨, 신규 크루 유치 경쟁은 지금 못 이김)

## 설계

### 1. 레이아웃 (`page-match`)

- "오늘의 매칭" 헤더 (카운트/희소성 노출 안 함)
- **히어로 카드 1장** — 기존 `#match-card` 재사용, **스와이프 제거**, **카드 탭 → 상세(`openUserProfile`)**
- 패스 / 같이 운동해요 버튼 (히어로 카드 대상)
- **"더 많은 추천" 썸네일 리스트** — 가로 스크롤, 탭 → 상세
- **진행바(`#match-progress`, "오늘의 추천 N/M") 제거**
- 후보 0명일 때: 빈 화면 대신 "같은 센터 둘러보기" 등 대체 동선/메시지

### 2. 데이터 소스 (Firestore `users`)

- `users` 컬렉션 로드
- 제외: 본인 / 이미 좋아요 보낸 사람(`likes` from==me) / 이미 매칭된 사람(`matches`) / 이번 세션 패스한 사람
- **정렬 우선순위: 같은 지역(구, `district`) → 같은 센터(`gym`) → 전체**
- 패스 = 이번 접속에서만 숨김 (메모리 `Set`, Firestore 저장 X). 사용자 많아지면 영구 저장으로 전환
- 같이 운동해요 = 기존 `window.sendLike(uid, name, photo)` 재사용 (likes/matches/알림 이미 처리됨)
- 히어로 카드 = 후보 배열의 현재 인덱스. 패스/좋아요 시 다음 후보로

### 3. 상세(`openUserProfile`) 매칭 전/후 차등

| 항목 | 매칭 전 | 매칭 후 |
|------|--------|--------|
| 사진 갤러리 | **전부 공개** (매칭률↑) | 전부 공개 |
| 동네(구) `district` | 공개 | 공개 |
| 운동 목적 `purpose` / 소개 `bio` | 공개 | 공개 |
| 이번달 인증횟수 `monthCnt` | 공개 (운동 진심 증명) | 공개 |
| **다니는 센터 `gym`** | 🔒 "매칭 후 공개" | 공개 |
| **운동 시간대 `workoutTime`** | 🔒 잠금 | 공개 |
| **운동 인증 피드** | 🔒 모자이크(현행) | 전체 공개 |
| 채팅 | 🔒 | 열림(채팅하기) |
| 버튼 | 같이 운동해요(`sendLike`) | 채팅하기(`openChatWith`) |

핵심: **"어디서·언제 만날 수 있나(센터+시간대)"를 매칭 후로 미룸.** 운동 맥락상 매칭=실제로 같이 운동하러 감 → 보상이 "만날 정보 해제"여야 자연스럽고, 프라이버시(낯선 사람에게 내 헬스장/운동시간 노출 방지)에도 안전.

## 구현 체크리스트 (순서)

1. `page-match` HTML: 스와이프 힌트 제거, 진행바 제거, "더 많은 추천" 리스트 컨테이너 추가, 카드에 탭 핸들러
2. 스와이프 JS(`_matchSwipeCommit`, swipe init) 제거/비활성
3. Firestore 후보 로더: `users` 로드 → 제외 필터 → 지역/센터 정렬 → 배열 보관
4. `renderMatch()` 재작성: 실유저 카드 렌더 + "더 많은 추천" 썸네일 렌더 + 빈상태 처리
5. `handleLike()`/`handlePass()` 재작성: 좋아요=`sendLike`, 패스=세션 Set에 추가 후 다음 후보
6. 카드/썸네일 탭 → `openUserProfile(uid, name, photo)`
7. `openUserProfile` 매칭 전 차등: `gym`·`workoutTime`을 not-matched일 때 잠금 표시
8. cache-bust 버전 업, git push

## 영향 범위

- 단일 파일 `index.html`

- 보안규칙 변경 없음 (패스 세션 전용이라 쓰기 없음)
