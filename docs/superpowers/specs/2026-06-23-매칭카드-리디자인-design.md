# 매칭 탭 카드 리디자인 (UI)

날짜: 2026-06-23
범위: `index.html` 매칭 탭(`#page-match`)의 카드 UI만. 데이터는 기존 더미 `PROFILES` 사용. Firestore 연동은 별도 후속 작업.

## 배경
현재 매칭 카드의 문제점:
- 위치('마포')가 이름 줄 우측에 떠서 사진/카드를 가림
- 정보 뱃지(이번달 N회 / 3대 N / 같은 센터) 배치가 정돈되지 않음
- '같이 운동해요' 버튼 아래 빈 공간이 어색함
- 카드 프레임이 머피 퍼스널 감성이 약함

## 결정된 방향 (브라우저 컴패니언으로 확정)

### 1. 카드 프레임 — "오버레이 정제" (B안)
- 풀블리드 사진 + 하단 진한 그라데이션 유지
- **위치 칩**: 이름 줄에서 분리 → 카드 **좌상단 글래스 칩**(`rgba(0,0,0,0.45)` + blur, radius 20px)
- **인증완료 뱃지**: 우상단 골드 (기존 위치 유지)
- 이름·나이 → 하단 좌측, 그 아래 운동 태그(`.tag`)
- **정보 뱃지 한 줄**: 이번달 N회 / 3대 N / 같은 센터
  - 기본 칩은 중립(흰색 반투명), '같은 센터'만 블루 강조(`rgba(61,126,255,0.18)` 배경 / `#7aa8ff` 텍스트)

### 2. 버튼 아래 빈 공간 — 진행 표시
- "오늘의 추천 N / M" 텍스트 + 가로 게이지 점들
- N = 현재까지 본 카드 수, M = 추천 풀 크기(더미: `PROFILES.length`)
- 패스/같이운동 시 N 증가, 게이지 채워짐

### 3. 스와이프 제스처
- 좌 드래그 = 패스, 우 드래그 = 같이 운동해요
- 드래그 중 카드 기울기(±7° 정도) + 방향 힌트 라벨("패스" 흰색 / "같이 운동" 블루)
- 임계값(약 90px) 넘겨 손 떼면 발동, 미만이면 원위치 복귀(스프링 전환)
- 기존 패스/같이운동 버튼 유지 → 스와이프·탭 둘 다 지원
- 터치(모바일) + 마우스 드래그(PC) 모두 대응

### 4. 아이콘 — 머피 라인 SVG
이모지 금지 원칙에 따라 전부 라인 SVG로:
- 위치: 핀 아이콘
- 인증: 방패 아이콘(기존 있음)
- 이번달: 캘린더 아이콘(기존 있음)
- 3대: 바벨 또는 번개 아이콘
- 스타일: `stroke="currentColor"`, stroke-width 1.7~2.2, 깔끔한 라인

## 구현 위치 (index.html)
- 정적 카드 마크업: `#page-match` (~506–543행)
- 렌더 로직: `renderMatch()` (~2122행), 오버라이드 `window.renderMatch` (~6085행)
- 액션: `handleLike()` / `handlePass()` (~2156행)
- 카드 CSS: `.match-card` / `.match-grad` / `.match-info` 등 (~145–150행)
- cache-bust 주석 버전 갱신 필수

## 비범위 (이번 작업 아님)
- Firestore 실데이터 연동, matches 컬렉션 저장
- 거리(km) 계산 — 위치 칩은 지역명만 표시
- 되돌리기/찜하기 보조 액션 (옵션 3 미채택)

## 확인 포인트
- 모바일 실기기에서 스와이프가 세로 스크롤과 충돌하지 않는지
- 짧은 드래그 후 원위치 복귀 자연스러운지
- 정보 뱃지 한 줄이 작은 화면에서 줄바꿈 시 깨지지 않는지
