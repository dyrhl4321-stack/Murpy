# 머피룸 월간 한정 오브젝트 시스템 — 설계

> 2026-07-21 브레인스토밍 확정. 대표 승인: "초안으로만 가면 됌" — 라인업·조건 수치는 초안 지위, 구현 중 대표 확인으로 확정.

## 목적

방 구경(mwVisitRoom)이 이미 열려 있고 머피 하이파이브가 예정된 상황에서, **방을 보여줄 이유·볼 이유**를 만든다. 매달 그 달에만 얻을 수 있는 한정 오브젝트로 챌린지 동기를 만들고, 방을 "언제부터 얼마나 활동했는지의 기록"으로 만든다.

## 확정된 제품 결정

1. **월 3~4종 구성**: 공개형 2~3종 + 완전 히든 1종 (B안+D안 혼합)
2. **공개형**: 도감에 실루엣 + 세계관 힌트 카피 노출. 정확한 조건 수치는 숨김. 힌트는 "숲의 요정들에게만 배달되는 선물입니다" 같은 머피 세계관 톤 (직설적 미션 문구 금지)
3. **완전 히든**: 존재 자체를 어디에도 표시하지 않음. 획득이 곧 첫 공개
4. **오브젝트 성격**: 기믹 가구·펫 중심 (공룡펫, 대왕 덤벨 등 "왜 이게 방에?" 류) + 메달·트로피 축. 획득 시 **칭호** 동반 부여
5. **진짜 한정판**: 월이 지나면 획득 경로 영구 소멸. 복각·재판매 없음. 도감에 지난 시즌 실루엣만 남김 ("다시는 만날 수 없습니다")
6. **획득 연출 (C안 혼합)**:
   - 공개형 → 조건 충족 즉시 축하 팝업 (기존 머피 적립 축하연출 패턴 재사용)
   - 히든 → 조용히 `pendingBox`에 적립 → 다음 내 방 입장 시 방에 상자 렌더 → 탭하면 개봉 연출. 안 뜯은 상자는 방문자에게도 상자로 보임
7. **판정: 클라이언트** (기존 크레딧·스트릭·도감보너스와 동일 신뢰 모델). Cloud Functions 이관은 공개 런칭 전 과제로 로드맵에만 기록

## 데이터 구조

**코드 상수 `SEASON_ITEMS`** (ROOM_ITEMS 옆): `{ id, name, src, w, h, flat?, season:'YYYY-MM', kind:'open'|'hidden', hint, title, cond }`
- `cond` 형식: `{ feed?: n, bamboo?: n, matchOk?: n, squad?: n, checkin?: n }` — 명시된 키 전부 충족(AND)
- 매달 운영 작업 = 에셋 제작 + 이 배열에 항목 추가

**`users/{uid}` 확장**:
- `monthly: { 'YYYY-MM': { feed, bamboo, matchOk, squad, checkin } }` — 활동 함수들에 `increment(1)` 한 줄씩 삽입
- `roomInv: [itemId]` — 획득 한정 오브젝트 (wardrobe 패턴)
- `titles: [string]` — 획득 칭호
- `pendingBox: [itemId]` — 미개봉 상자 (히든 전용)

## 판정·획득 흐름

```
활동 발생 → monthly['현재월'][축] increment → 클라 판정 함수
  ├─ 공개형 충족(미보유) → 즉시 축하 팝업 + roomInv·titles 갱신
  └─ 히든 충족(미보유) → pendingBox 추가 (무통지)
        → 내 방 입장 시 상자 렌더 → 탭 → 개봉 연출 → pendingBox→roomInv 이동
월 변경 시: 현재 월 시즌만 판정. 지난 시즌 조건 판정 중단(영구 소멸)
```

## UI

1. **도감 "이번 달" 섹션**: 공개형 실루엣(grayscale/실루엣 처리) + 힌트 카피 + 남은 일수. 획득 시 컬러 전환. 히든은 미표시, 획득자만 "???" 슬롯 컬러 등장
2. **지난 시즌 아카이브**: 실루엣 + "다시는 만날 수 없습니다"
3. **방 편집기**: roomInv 보유분이 카탈로그 하단 "한정" 섹션으로 추가. 배치 로직은 기존 `_redit*` 그대로
4. **칭호 표시**: 방 구경 패널·방명록 닉네임 옆 최근 획득 칭호 1개 자동 표시 (선택 UI 없음)

## 첫 시즌 (2026-08) 라인업 — 초안 (에셋 제작 전 대표 확인)

| 종류 | 오브젝트 | 축 | 힌트 카피 초안 |
|---|---|---|---|
| 공개 | 대왕 덤벨 | 인증(feed) | "헬스장 지박령의 기운이 느껴지는 자에게" |
| 공개 | 숲 요정 램프 | 대숲(bamboo) | "숲의 요정들에게만 배달되는 선물입니다" |
| 공개 | 스쿼드 캠프파이어 | 스쿼드(squad) | "혼자 온 자는 받을 수 없습니다" |
| 히든 | 아기 공룡 | 복합(비밀) | 없음 |

조건 수치는 베타 규모(GBD 초기 인원) 감안해 낮게 시작, 구현 중 확정.

## 범위 제외

- 칭호 선택·관리 UI
- 오브젝트 애니메이션 (정적 PNG, 공룡 포함)
- 서버 판정 (Cloud Functions)
- 복각·재판매

## 기존 코드 접점

- `ROOM_ITEMS` / `mwFurnHtml` / `_redit*` (index.html ~1990행대): 방 카탈로그·렌더·편집
- `mwVisitRoom` (~10272행): 방 구경 — 상자 렌더·칭호 표시 지점
- `_creditState`·크레딧 적립(~7033행대): 축하연출 재사용·monthly 카운터 삽입 지점과 인접
- 활동 함수: `submitFeedPost`, `postBambooFirestore`, `_acceptBambooMatch`/매칭 성사, 스쿼드 출석, 센터 체크인 — 각각 monthly increment 삽입
- 에셋: `char/rooms/` + 기존 누끼→픽셀 파이프라인, sw.js 캐시 버전 갱신 필수
