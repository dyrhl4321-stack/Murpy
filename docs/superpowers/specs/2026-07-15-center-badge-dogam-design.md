# 센터 픽셀뱃지 · 도감 개편 · 우리센터머피들 · SNS 공유 · 공문 광고 이미지 — 설계

날짜: 2026-07-15
목적: 헬스장에 보낼 "로고 픽셀화 협조 공문"에 첨부할 종합 광고 이미지를 **실제 앱 화면 기반**으로 만들기 위해, 레퍼런스(클코에게 보내줄 로고픽셀뱃지 변환예시.png)의 UI를 앱에 실구현한다.
레퍼런스: `Desktop/김현수 컴카드/머피브랜딩/머피월드 캐릭터/커스터마이징 3차/헬스장로고/헬스장로고공문파일/` 2장.

## 확정 결정사항
1. **뱃지 = 픽셀로고(센터별 AI 변환) + 공용 프레임(금/은 각 1장) 코드 합성** — 뱃지 통짜 생성 아님.
2. **도감 보너스 = 서로 다른 센터 5곳 +20 / 10곳 +50 / 20곳 +100 크레딧, 각 1회성.**
3. **우리센터머피들 진입 = 지도 센터그룹 클릭 + 센터 상세 버튼 (둘 다).**
4. **필드 연출 = 고정배치 + 제자리 걷기 애니** (돌아다니기 시뮬레이션 아님).
5. **광고 이미지 = HTML 조판 + 실앱 캡처, 가로 1장 + 세로 1장.** 과장 허용(인원·도장 수 연출).
6. 파동함수 붕괴식 센터별 인테리어(LimeZu)는 **범위 밖** — 필드 이미지 1장 공통, 교체 가능한 구조만 준비.

## 1. 에셋 파이프라인
- **제미나이(나노바나나) 프롬프트 3종 작성** (산출물: 프롬프트 텍스트, 대표/사용자가 생성 → 내가 후처리):
  1. 금테 프레임: 투명배경, 원형 픽셀 링, 레퍼런스 AFTER의 금테 질감(하이라이트·그림자 스텝 셰이딩). 은테는 같은 형태 컬러 스왑 프롬프트.
  2. 로고 픽셀화(표준 재사용): 첨부 로고를 저해상 픽셀 아트로 재해석, 배경/테두리 없이 로고만, 팔레트 축소.
  3. 와이드 헬스장 필드: 업다운뷰(탑다운) 픽셀 헬스장 실내, 가로형(16:9급), 기구 배치, 캐릭터 없는 빈 필드.
- 후처리(내 작업): 누끼·팔레트 정리·NEAREST 정수배 리사이즈 →
  - `char/ui/badge_frame_gold.png`, `char/ui/badge_frame_silver.png` (같은 캔버스 규격, 예: 96×96 링 + 내부 투명)
  - `char/fields/field_gym_wide.png`
- **라이크짐 픽셀로고**: 공문파일 PNG의 AFTER 뱃지에서 로고 영역 추출·정리(금테 제거, 로고만) → imgBB 업로드 → 라이크짐 센터 문서 `pixelLogo`에 입력. 품질 미달이면 프롬프트 2로 재생성.
- 하드룰 준수: NEAREST·정수배만, base 시트 무관(캐릭터 시트 안 건드림).

## 2. 뱃지 시스템 (index.html)
- 데이터: `centers.pixelLogo`(string URL, 선택) 추가. 금/은은 기존 `partner`(bool)로 결정.
- 관리자 센터편집 패널에 "픽셀 로고 URL" 입력칸 추가(기존 `logo` 필드와 별개 유지).
- 공용 렌더 `mwBadgeHtml(center, size)` / 캔버스 합성 `_mwDrawBadge(canvas, center)`:
  - 프레임: partner→gold, 아니면 silver. 프레임 PNG를 캔버스에 드로우(imageSmoothingEnabled=false).
  - 내부: `pixelLogo` 있으면 그대로(정수배), 없으면 **기존 12px 자동픽셀화 폴백**(현 `_mwPixelize` 로직 통합), 그것도 없으면 이니셜 타일.
- 기존 `mw-pixlogo` 사용처(도감 셀)는 뱃지 렌더로 교체.

## 3. 도감 개편
- 셀 UI: 위 센터명 / 가운데 뱃지 / 아래 "방문 N회" (레퍼런스 02). 미방문=? 잠금 유지. 지역 아코디언·홈센터 카드·칭호 시스템은 불변.
- 도감 보너스 바(도감 하단 고정 패널):
  - 표기: "도감 보너스 — 5곳 +20 · 10곳 +50 · 20곳 +100" + 현재 진행(예: 7/10) 프로그레스.
  - 판정 순수함수 `dogamBonusDue(distinctCount, claimed)` → 지급할 단계 배열 반환. node 테스트 작성.
  - 지급: `users.dogamBonus = {b5:true,b10:true,b20:true}` 필드로 중복 방지, `mwUpdateCoin`으로 적립 + 기존 코인 축하연출 재사용. 체크인 완료 시점(`mwSealCheckin`)에 판정.
  - distinctCount = 내 checkins의 서로 다른 centerId 수.
- CREDITS_MODE='beta' 그대로 (적립만).

## 4. 우리 센터 머피들 (`mwCenterField(centerId)`)
- 진입: ① 지도 패널 센터 그룹 헤더 클릭 ② 센터 상세에 "우리 센터 머피들 N명" 버튼.
- 화면(풀스크린 오버레이): 상단 뱃지+센터명+"현재 체크인 N명", 본문 `field_gym_wide.png` 배경 위에 오늘 체크인 머피들.
- 캐릭터: `mwMiniCharHtml` 재사용, 높이 ~44px. 위치는 `hash(uid+day)` 시드로 필드 내 안전영역 그리드에 배치(겹침 방지 지터). 제자리 걷기 = CSS steps() background-position 애니. 닉네임 라벨. 캐릭터 탭 → 그 머피 방 방문(기존 mwVisitRoom).
- 필드 이미지는 `centerFieldSrc(center)` 함수로 분리 — 추후 센터별 이미지로 교체 가능.
- 0명일 때: 빈 필드 + "오늘 첫 도장의 주인공이 되어보세요".

## 5. SNS 공유 (스탬프 결과 화면)
- 버튼 3개: **인스타**(navigator.share files — 공유시트에서 인스타 선택; 미지원 시 저장 폴백) / **카톡**(Kakao.Share.sendDefault feed 템플릿, imageUrl=imgBB 업로드 URL, link=앱 URL; 자유촬영이면 공유 시점에 업로드) / **저장**(현행 mwSaveStamp).
- Kakao 공유는 카카오 콘솔 도메인 등록 상태에 의존 — 로그인용으로 이미 등록돼 있을 것으로 예상, 실기기 확인 항목.

## 6. 공문용 광고 이미지 (최종 산출물)
- 연출 캡처: 관리자 테스트 체크인·연출 계정 데이터로 도감(뱃지 여러 개)·우리센터머피들(다수 인원)·스탬프 화면을 실앱에서 캡처. 과장 허용.
- `promo/center-pitch-landscape.html`(가로 3:2, 2400×1600급) + `promo/center-pitch-portrait.html`(1080×1920급): 레퍼런스 5섹션(①변환예시 Before→After ②도감 ③우리센터머피들 ④스탬프+SNS ⑤일반vs제휴 혜택) 조판. 변환예시=라이크짐.
- PNG 추출 → `헬스장로고공문파일/` 폴더에 저장. 공문 본문 텍스트는 대표 기존안 사용(범위 밖).

## 7. 검증·배포
- 순수함수 node 테스트(도감보너스 판정, 배치 시드). 뱃지·필드는 합성 미리보기 이미지로 정렬 증명 후 등록(검수 원칙).
- 배포: push + sw 버전 업. 대표 실앱 확인(체크인·공유는 실기기 필요).

## 범위 밖
- LimeZu 센터별 인테리어(파동함수 붕괴), 크레딧 차감(enforced), 공문 문서 자체 작성, 인스타 정식 API(존재하지 않음 — 공유시트가 표준).
