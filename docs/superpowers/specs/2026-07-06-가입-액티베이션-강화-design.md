# 가입 액티베이션 강화 설계 (콜드스타트 스펙 1)

- 작성일: 2026-07-06

- 상위 맥락: 머피월드 고도화 → 콜드스타트. **스펙1=가입 액티베이션(먼저)**, 스펙2=머피월드 게임식 퀘스트판(다음, 별도).

## 1. 목표 (What & Why)

신규 유저가 가입 직후 **① 프로필을 확실히 채우고 ② 머피(코인)의 가치를 이해한 뒤 ③ 핵심 기능(매칭·대숲·피드)으로 자연스럽게 진입**하도록 온보딩/액티베이션을 강화한다.

**핵심 원칙(대표 지시):**
- 핵심 기능 = **매칭·피드·대숲.** 머피/머피월드는 이를 붙잡는 **연결고리·서브월드.**
- 가입 직후 **머피월드로 강제 이동 절대 금지.** "지금 머피월드 가실래요?" 류 X.
- 머피월드는 **아주 미세하게(한 줄)** 만 흘리고, 유저는 핵심 기능에 안착.

## 2. 현재 상태 (코드 실측)

- 온보딩 모달 `#onboarding-modal`: 2스텝. Step1(닉네임·성별·나이, `obNextStep` 7361 근처 검증됨), Step2(운동목적·시간대·지역·주종목*·소개, `obComplete` `:7361`). **프로필 사진 단계 없음.**
- `obComplete`(`:7361`): 주종목·지역 필수 검증 → users 저장(`onboardingComplete:true`) → 모달 닫고 토스트 "머피에 오신 걸 환영해요!". **머피 축하/설명 없음.**
- 머피 가입 지급: `ensureCreditsInit(uid)`(`:6218`)가 **조용히** 지급(+10). 연출 없음.
- 축하 연출 `celebrateMurpy(balance)`(`:5936`, `#murpy-celebrate` `:2973`): 코인 회전 짜잔. **이미 있음.**
- 설명 패널 `#murpy-first-guide`(`:2984`) + `murpyFirstGuide()`(`:5975`): "머피는 **매칭 신청·대숲 글쓰기**에 써요" + [매칭 보러가기]/[대숲 가기]/[닫기]. **이미 있음.**
- 현재 이 축하+가이드는 **첫 운동 인증**(`submitFeedPost` `:7480`, `grantDailyCreditOnVerify` 성공 시)에서만 뜸. `localStorage 'murpy_credit_intro'`로 1회 게이트.
- 프로필 사진 크롭: `confirmProfileCrop()`(`:4736`), 크롭 모달(`:896~`).

## 3. 설계

### 3.1 온보딩에 프로필 사진 단계 추가 (필수)

- 기존 2스텝 뒤에 **Step 3 = 프로필 사진**. 진행 닷 2개 → **3개**로.
- Step3 UI: 안내문 + 사진 선택 버튼(원형 크롭 미리보기) + "머피 시작하기" 버튼. 기존 크롭 플로우(`confirmProfileCrop`) 재활용해 `users.photoURL` 저장.
- **게이트: 사진 없으면 "머피 시작하기" 불가.** (사진 미등록 시 토스트 "프로필 사진을 등록해주세요" 후 중단)
- Step2의 "머피 시작하기" 버튼 → **"다음 →"**(Step3로 이동)으로 변경. 최종 완료 버튼은 Step3에 위치.
- 기존 검증 유지: 닉네임 2자+, 성별, 나이 15~80(Step1) / 주종목·지역(Step2/완료).

### 3.2 머피 지급 액티베이션 = 축하+가이드를 "가입 완료" 시점으로 이동(재활용)

- `obComplete` 성공 직후(users 저장·모달 닫은 뒤): **`celebrateMurpy(10)` → 이어서 `murpyFirstGuide()`** 호출.
  - `submitFeedPost`의 패턴(`:7481-7485`)을 가입 시점으로 옮김. `localStorage 'murpy_credit_intro'`를 **가입 때 세팅** → 첫 운동 인증 때는 가이드 재노출 안 함(축하 연출은 인증 때도 유지 가능).
- **`#murpy-first-guide` 문구 = 가입 선물 톤으로 소폭 수정:**
  - 제목: "머피가 쌓이기 시작했어요" → **"머피 10개를 받았어요 🎁"**
  - 본문 유지+보강: "머피는 **매칭 신청**과 **대숲 글쓰기**에 써요." + **아주 작게 한 줄**: "꾸민 캐릭터로 노는 **머피월드**에서도 써요."(머피월드 강조 X, 미세 노출)
  - 버튼 유지: **[매칭 보러가기]**(→match) / **[대숲 가기]**(→bamboo) / [닫기]. → 핵심 기능으로 유도(머피월드 버튼 없음). "닫기"는 홈에 안착.
- 즉 **새 팝업 안 만들고** 기존 축하+가이드를 재활용·재배치·문구 수정.

### 3.3 흐름

```
가입(구글/카카오) → 온보딩
  Step1 닉/성별/나이 → Step2 운동정보 → Step3 프로필사진(필수)
→ "머피 시작하기"(obComplete)
→ celebrateMurpy(10) 코인 짜잔
→ #murpy-first-guide (머피 10 선물톤 + 매칭/대숲 유도 + 머피월드 한 줄)
→ [매칭 보러가기]/[대숲 가기]/[닫기] → 핵심 기능/홈에 안착
```
**머피월드로 강제 이동 없음.**

## 5. 검증

로컬(`python -m http.server`)에서 **신규 유저 시뮬**:
- 온보딩 Step1→2→3 진행, **Step3에서 사진 없이 완료 시도 → 막힘**, 사진 등록 후 완료.
- 완료 직후 코인 축하 → 가이드 패널(선물톤·매칭/대숲 버튼·머피월드 한 줄) 뜨는지.
- [매칭 보러가기]→매칭, [대숲 가기]→대숲, [닫기]→홈 안착. **머피월드로 안 튐** 확인.
- 재로그인/기존 유저는 온보딩·가이드 재노출 안 됨(`onboardingComplete`, `murpy_credit_intro`).
- `node --check` 인라인 스크립트.
