# 크레딧 적립 "번개 적립" 중앙 축하 연출 — 디자인

- 날짜: 2026-06-26
- 상태: 승인됨 (구현 대기)
- 관련: `2026-06-25-운동크레딧-시스템-design.md` (크레딧 시스템 본체)

## 1. 배경 / 문제

피드 글(운동 인증) 작성 완료 시 하루 1회 운동 크레딧(`CREDIT_DAILY = 1`)이 적립된다. 현재 연출은:

- 상단 토스트: `✓ 운동 인증 완료 · +1 크레딧!` (매일 첫 적립마다, `index.html` 6222)
- 평생 첫 적립 때만 `openCreditInfo(true)` 호출 → **하단에서 올라오는 `.modal-sheet`** (6223)

문제: 이 하단 시트는 **프로필에서 크레딧 카드를 탭할 때 뜨는 UX와 완전히 동일**(`openCreditInfo` 재사용)하다. 적립 순간과 정보 조회가 같은 연출이라 "중복 UX"로 보이고, 적립의 특별함이 약하다. 목표는 적립 순간을 **중앙 축하 연출**로 분리해 "제대로 만든 앱" 느낌을 주되, 매일 봐도 질리지 않게 절제하는 것.

## 2. 목표 / 비목표

**목표**
- 피드 작성으로 크레딧이 실제 적립되는 순간을 **중앙 "번개 적립" 축하**로 연출 (토스/핀테크 톤).
- 프로필 크레딧 UX(하단 시트)와 **중복 제거**.
- 매일 1회 빈도, 절제된 모션으로 **반복해도 안 질리게**.

**비목표**
- 크레딧 적립 로직 자체(`grantDailyCreditOnVerify`, 금액, 주기)는 변경하지 않는다.
- 매칭/대숲 크레딧 소모(enforced) 흐름은 손대지 않는다.
- 피드 2페이지 스크롤 버그는 이 스펙 범위 밖(별도 직접 수정).

## 3. 트리거 & 빈도

- 피드 작성 제출(`index.html` ~6218) 후 `grantDailyCreditOnVerify(uid)` 결과가 **`true`(오늘 첫 적립)** 일 때만 중앙 축하를 띄운다.
- 같은 날 추가 글(`false`): 축하 없이 기존 조용한 토스트 `✓ 올라갔어요!` 유지.
- 크레딧 미적립/비로그인: 변화 없음.

## 4. 연출 — 번개 적립 (모션 타임라인)

배경/대상: 다크 테마, 크레딧 보라(#A78BFA), 강조 골드(#F5C24B). **이모지 미사용**, 머피 라인 SVG 아이콘만 사용(번개 아이콘은 기존 `path d="M13 2 4 14h7l-1 8 9-12h-7l1-8z"` 재사용).

1. `0ms`: 배경 오버레이 페이드인 — `rgba(0,0,0,0.55)` + `backdrop-filter: blur(6px)` (~150ms)
2. `100ms`: 중앙 번개 아이콘 스프링 등장 — `scale 0 → 1.1 → 1` (ease-out, ~360ms), 뒤 방사형 글로우 1회 펄스(opacity/scale)
3. `260ms`: "+1" 숫자 `0 → 1` 카운트업 + 살짝 팝
4. `300ms`: 골드 스파클 SVG 3~4개가 아이콘 주변에서 위로 흩어지며 페이드아웃 (~700ms)
5. 텍스트: 제목 **"운동 크레딧 +1"**, 보조 **"이제 N개"** (N = 적립 후 잔액)
6. `1.8s` 경과 또는 **아무 곳 탭** → 페이드아웃 + `scale 1 → 0.96` (~220ms) 후 DOM 숨김
7. `@media (prefers-reduced-motion: reduce)`: 모든 transform/스파클 제거, **단순 페이드인/아웃**만.

상호작용: 표시 중 중복 호출 방지(이미 떠 있으면 무시/리셋). 탭 시 즉시 닫기. 자동닫힘 타이머는 탭 시 정리.

## 5. 후속 안내 (평생 첫 적립만)

- 조건: `localStorage.murpy_credit_intro` 미설정.
- 동작: 중앙 축하가 닫힌 직후(약 220ms 후) **중앙 톤의 안내 카드**가 자연스럽게 이어짐.
  - 내용: "크레딧은 **매칭 신청**·**대숲 글쓰기**에 써요" + 버튼 [매칭 보러가기] [대숲 가기] + [닫기]
  - 이 안내는 **하단 시트(`openCreditInfo`)가 아니라** 같은 중앙 오버레이 톤으로 신규 제작.
  - 표시 후 `localStorage.murpy_credit_intro = '1'` 기록 → 이후로는 매일 축하만.
- 그 외(이미 첫 적립 경험): 후속 없음, 축하만.

## 6. 중복 제거 — 변경 지점 (현 코드 기준)

- `index.html` 6222: 적립 케이스의 상단 토스트 **생략**(중앙 축하로 대체). 비적립 케이스만 `✓ 올라갔어요!` 토스트 유지.
- `index.html` 6223: 첫 적립 시 `openCreditInfo(true)` 호출 **제거** → `celebrateCredit()` + (첫적립 시) `creditFirstGuide()` 로 대체.
- `openCreditInfo` 함수 및 `.modal-sheet`(`credit-info-modal`)은 **프로필 크레딧 카드 탭 경로(`openCreditInfo(false)`)에서만** 유지. `celebrate=true` 분기는 더 이상 피드 흐름에서 호출되지 않음(함수 자체는 보존하되 호출부 제거).

## 7. 구현 형태 (단일 HTML 패턴 준수)

- **DOM**: 오버레이 1개 신규 추가 — `#credit-celebrate` (고정 전체화면, 중앙 정렬, 기본 `display:none`). 내부에 번개 아이콘/글로우/`+1`/제목/잔액/스파클 컨테이너.
- **CSS**: `@keyframes`로 스프링 등장·글로우 펄스·스파클 드리프트·페이드 정의. 기존 색상 변수/클래스 톤 재사용.
- **JS**: 모듈 스코프 → `window` 브릿지 패턴 준수.
  - `window.celebrateCredit(balance)`: 오버레이 표시·애니메이션·카운트업·자동닫힘.
  - `creditFirstGuide()`: 첫 적립 안내 카드 표시(중앙 톤). 신규 함수.
  - 피드 제출부에서: `const earned = await grantDailyCreditOnVerify(...)` → `if (earned) { window.celebrateCredit(window.getCredits()); if (첫적립) creditFirstGuide(); } else { showToast('✓ 올라갔어요!'); }`
- **캐시버전**: 변경 시 `index.html` cache-bust 주석 및 `sw.js` 캐시 버전 1단계 상향.

## 8. 접근성 / 안전

- `prefers-reduced-motion: reduce` 대응(페이드만).
- 키보드/탭 어디서나 닫힘, 자동닫힘 타이머 누수 없음.
- 적립 실패/네트워크 오류 시 축하 미표시(기존 토스트 흐름 유지).
- 중복 호출·연타 방어.

## 9. 검증 방법

- 로컬 서버(`python -m http.server`)에서 새 계정/`murpy_credit_intro` 비운 상태로:
  1. 오늘 첫 인증 → 중앙 번개 축하 + (첫 적립이면) 안내 카드.
  2. 같은 날 두 번째 글 → 축하 없음, `✓ 올라갔어요!` 토스트만.
  3. 프로필 크레딧 카드 탭 → 기존 하단 시트 정상(중앙 축하와 분리됨).
  4. reduced-motion 설정 시 페이드만.
- 인라인 스크립트 문법 검사 통과(앱 로드 무결성).

## 10. 범위 외 (별도 처리)

- **피드 2페이지(태그 아래) 스크롤 버그**: 친구 계정에서 재현 확인됨. 본 스펙과 무관한 결함으로, 구현 단계에서 원인 진단 후 직접 수정한다.
