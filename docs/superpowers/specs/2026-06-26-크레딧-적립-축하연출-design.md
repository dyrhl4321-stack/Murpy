# 머피 적립 "m-토큰" 중앙 축하 연출 + 화폐명 통일 — 디자인

- 날짜: 2026-06-26
- 상태: 승인됨 (구현 대기)
- 관련: `2026-06-25-운동크레딧-시스템-design.md` (크레딧 시스템 본체)

## 1. 배경 / 문제

피드 글(운동 인증) 작성 완료 시 하루 1회 크레딧(`CREDIT_DAILY = 1`)이 적립된다. 현재 연출은:

- 상단 토스트: `✓ 운동 인증 완료 · +1 크레딧!` (매일 첫 적립마다, `index.html` 6222)
- 평생 첫 적립 때만 `openCreditInfo(true)` 호출 → **하단에서 올라오는 `.modal-sheet`** (6223)

문제 1 (중복): 이 하단 시트는 **프로필에서 크레딧 카드를 탭할 때 뜨는 UX와 완전히 동일**(`openCreditInfo` 재사용)해서 적립의 특별함이 약하다.

문제 2 (정체성): 보상 단위가 일반적인 "크레딧"이고 아이콘이 번개라, (a) 브랜드 정체성이 없고 (b) 번개가 "오늘 뜨는 컨텐츠"(불꽃 계열)와 에너지 느낌이 겹친다.

## 2. 목표 / 비목표

**목표**
- 보상 단위를 **"머피"**로 명명(브랜드=화폐). 예: "1머피 적립", "이제 12머피", "매칭에 1머피 사용".
- 심볼을 **머피 m-마크 토큰**(보라 발광 칩)으로 교체 — 기존 번개 아이콘 대체.
- 적립 순간을 **중앙 "m-토큰 적립" 축하**로 연출(토스/핀테크 톤), 프로필 하단 시트와 **중복 제거**.
- 매일 1회 빈도, 절제된 모션으로 **반복해도 안 질리게**.

**비목표**
- 적립 로직(`grantDailyCreditOnVerify`, 금액, 주기)과 내부 데이터(Firestore `credits` 필드, `CREDITS_MODE`, 상수명)는 **변경하지 않는다**. 화면에 보이는 단어/아이콘만 바꾼다.
- 매칭/대숲 소모(enforced) 흐름 자체는 손대지 않는다(문구만 "머피"로).
- 피드 2페이지 스크롤 버그는 이 스펙 범위 밖(별도 직접 수정).

## 3. 화폐명 통일: "크레딧" → "머피" (사용자 문구 전부)

내부 식별자(`credits`, `creditsLastEarned`, `CREDIT_DAILY`, `getCredits`, `#credit-amount` 등 id/변수)는 그대로 두고, **사용자에게 보이는 텍스트만** 교체한다. 대상(현 코드 기준):

| 위치 | 현재 | 변경 후 |
|---|---|---|
| 1326 | 운동하면 크레딧이 쌓여요 | 운동하면 머피가 쌓여요 |
| 1327 | 크레딧 1개 / 크레딧으로 매칭 신청과 대숲 글쓰기 | 1머피 / 머피로 매칭 신청과 대숲 글쓰기 |
| 2205 | 축하해요! 크레딧이 쌓였어요 / 운동하면 크레딧이 쌓여요 | (하단 시트 비축하 문구) 운동하면 머피가 쌓여요 |
| 5093 | 이 신청에 크레딧 N개를 사용해요 | 이 신청에 N머피를 사용해요 |
| 5095 | 크레딧이 부족해요 · 오늘 운동 인증하면 바로 써요 | 머피가 부족해요 · 오늘 운동 인증하면 바로 써요 |
| 6407 | 크레딧이 부족해요 · … | 머피가 부족해요 · … |
| 6222 | ✓ 운동 인증 완료 · +N 크레딧! | (적립 시 중앙 축하로 대체, §6) |

추가: **번개 SVG 아이콘을 머피 m-토큰으로 교체**할 위치 — 프로필 크레딧 카드(732~736 영역), 크레딧 안내 모달(1324 영역), 중앙 축하(신규). 단위 표기는 "N머피"(붙여 씀)로 통일.

## 4. 심볼: 머피 m-토큰

- 기반: 앱 아이콘 `icon.svg`의 **흰 원 + 3-아치 m 레터마크**.
- 토큰 형태: 원형 칩 안에 m-마크. 적립/축하 맥락에선 **보라(#A78BFA) 발광**, 강조는 골드(#F5C24B) 스파클.
- 라인 SVG 스타일 유지, **이모지 미사용**.

## 5. 트리거 & 빈도

- 피드 작성 제출(`index.html` ~6218) 후 `grantDailyCreditOnVerify(uid)` 결과가 **`true`(오늘 첫 적립)** 일 때만 중앙 축하.
- 같은 날 추가 글(`false`): 축하 없이 기존 조용한 토스트 `✓ 올라갔어요!`.
- 미적립/비로그인: 변화 없음.

## 6. 연출 — m-토큰 적립 (모션 타임라인)

배경/대상: 다크 테마, 머피 보라(#A78BFA), 강조 골드(#F5C24B).

1. `0ms`: 배경 오버레이 페이드인 — `rgba(0,0,0,0.55)` + `backdrop-filter: blur(6px)` (~150ms)
2. `100ms`: 중앙 **m-토큰 칩이 스프링 등장** — `scale 0 → 1.1 → 1`(ease-out, ~360ms), 살짝 회전(예: -8° → 0°), 뒤로 부드러운 방사형 글로우 1회 펄스
3. `260ms`: **"+1머피"**의 숫자 `0 → 1` 카운트업 + 살짝 팝
4. `300ms`: 골드 스파클 SVG 3~4개가 칩 주변에서 위로 흩어지며 페이드아웃(~700ms)
5. 텍스트: 제목 **"1머피 적립!"**, 보조 **"이제 N머피"** (N = 적립 후 잔액)
6. `1.8s` 경과 또는 **아무 곳 탭** → 페이드아웃 + `scale 1 → 0.96`(~220ms) 후 DOM 숨김
7. `@media (prefers-reduced-motion: reduce)`: transform/스파클 제거, **단순 페이드인/아웃**만.

상호작용: 표시 중 중복 호출 방지(리셋), 탭 시 즉시 닫기, 자동닫힘 타이머 정리.

## 7. 후속 안내 (평생 첫 적립만)

- 조건: `localStorage.murpy_credit_intro` 미설정.
- 동작: 중앙 축하가 닫힌 직후(약 220ms 후) **중앙 톤의 안내 카드**가 자연스럽게 이어짐.
  - 내용: "머피는 **매칭 신청**·**대숲 글쓰기**에 써요" + 버튼 [매칭 보러가기] [대숲 가기] + [닫기]
  - **하단 시트(`openCreditInfo`)가 아니라** 같은 중앙 오버레이 톤으로 신규 제작.
  - 표시 후 `localStorage.murpy_credit_intro = '1'` 기록 → 이후로는 매일 축하만.
- 그 외: 후속 없음, 축하만.

## 8. 중복 제거 — 변경 지점 (현 코드 기준)

- `index.html` 6222: 적립 케이스 상단 토스트 **생략**(중앙 축하 대체). 비적립만 `✓ 올라갔어요!`.
- `index.html` 6223: 첫 적립 시 `openCreditInfo(true)` 호출 **제거** → `celebrateMurpy()` + (첫적립 시) `murpyFirstGuide()` 로 대체.
- `openCreditInfo`/`.modal-sheet`(`credit-info-modal`)은 **프로필 크레딧 카드 탭(`openCreditInfo(false)`)에서만** 유지. `celebrate=true` 분기는 피드 흐름에서 더 이상 호출되지 않음(함수 보존, 호출부 제거).

## 9. 구현 형태 (단일 HTML 패턴 준수)

- **DOM**: 오버레이 1개 신규 — `#murpy-celebrate`(전체화면, 중앙 정렬, 기본 `display:none`). 내부에 m-토큰 칩/글로우/`+1머피`/제목/잔액/스파클.
- **CSS**: `@keyframes`로 스프링 등장·회전·글로우 펄스·스파클 드리프트·페이드. 기존 색상 톤 재사용.
- **JS**(모듈 → `window` 브릿지):
  - `window.celebrateMurpy(balance)`: 오버레이 표시·애니메이션·카운트업·자동닫힘.
  - `murpyFirstGuide()`: 첫 적립 중앙 안내 카드(신규).
  - 피드 제출부: `const earned = await grantDailyCreditOnVerify(...)` → `if (earned){ window.celebrateMurpy(window.getCredits()); if(첫적립) murpyFirstGuide(); } else { showToast('✓ 올라갔어요!'); }`
- **재사용 가능한 m-토큰 SVG**: 헬퍼(예: `murpyTokenSVG(size)`)로 만들어 카드/모달/축하에서 공통 사용.
- **캐시버전**: `index.html` cache-bust 주석 및 `sw.js` 캐시 버전 1단계 상향.

## 10. 접근성 / 안전

- `prefers-reduced-motion: reduce` 대응(페이드만).
- 어디서나 탭 닫힘, 타이머 누수 없음.
- 적립 실패/네트워크 오류 시 축하 미표시(기존 흐름 유지).
- 중복 호출·연타 방어.

## 11. 검증 방법

- 로컬 서버에서 새 계정/`murpy_credit_intro` 비운 상태:
  1. 오늘 첫 인증 → 중앙 m-토큰 축하("1머피 적립!") + (첫 적립이면) 안내 카드.
  2. 같은 날 두 번째 글 → 축하 없음, `✓ 올라갔어요!`만.
  3. 프로필 크레딧 카드 탭 → 기존 하단 시트 정상(축하와 분리), 문구·아이콘은 "머피"/m-토큰.
  4. 매칭/대숲 사용·부족 안내 문구가 "머피"로 통일됐는지.
  5. reduced-motion 설정 시 페이드만.
- 인라인 스크립트 문법 검사 통과(앱 로드 무결성).

## 12. 범위 외 (별도 처리)

- **피드 2페이지(태그 아래) 스크롤 버그**: 친구 계정에서 재현 확인됨. 본 스펙과 무관한 결함으로, 구현 단계에서 원인 진단 후 직접 수정한다.
