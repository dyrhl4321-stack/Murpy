# 비로그인 액티베이션 (소비형 아하 모먼트) — 설계

작성일: 2026-06-25
대상 파일: `index.html` (단일 HTML)

## 목표

비로그인 방문자를 **빠르게 아하 모먼트로 연결**해 가입 전환(액티베이션)을 높인다.
운동 인증 "게시"는 로그인 뒤에 두되(보안규칙·작성자 귀속을 건드리지 않음),
비로그인은 **소비형 경험**으로 끌어당긴다.

## 확정된 방침

- **게시는 풀지 않는다.** 비로그인 익명 글쓰기 없음 → Firestore 보안규칙·작성자 귀속 그대로.
- **두 레버를 함께 쓴다:** (1) 실시간 활기 강조 + (2) 고의도 순간의 매끄러운 JIT 로그인 전환.
- **콜드스타트 정직성:** 가짜 카운트·더미 금지. 베타 초기 "오늘 1~2명" 같은
  김 빠지는 숫자를 노출하지 않는다. 숫자는 지어내지 않고, 실제 데이터 중 가장
  정직하면서 살아있어 보이는 범위를 고른다.
- **UI 제약:** 이모지 금지 → 머피 전용 라인 SVG 아이콘만. 색상 블랙/화이트 + 블루(#3D7EFF)/골드(#F5C24B).

## 현재 자산 (재사용)

- `renderHomeLiveBar()` (index.html:4689): 오늘 인증 고유유저 집계 + 아바타 + 불꽃 SVG.
  단, 오늘 데이터가 비면 숨김 → 콜드스타트 약점.
- `requireLogin(reason)` (index.html:2148) → `login-guard-modal` (index.html:1278):
  **이미** 구글·카카오 1탭 버튼 + 맥락 텍스트(`login-guard-text`)를 갖춘 JIT 로그인 시트.
- 비로그인도 `loadFeed()` 실행되어 피드를 봄 (index.html:4583 부근).
- 사진 인증 차단 지점: index.html:3509 — 비로그인이 사진을 **고른 뒤** 입력 초기화 + `requireLogin`.

## 설계

### 1. 활기 — 콜드스타트 안전한 라이브바 (3단 폴백)

`renderHomeLiveBar()`를 다음 우선순위로 변경한다. 숫자는 모두 실제 집계.

1. **오늘** 인증 고유유저 수 ≥ `LIVE_MIN`(= 3) → "오늘 N명이 운동 인증했어요" (기존 동작)
2. 아니면 **최근 7일** 집계 ≥ `LIVE_MIN` → "이번 주 N명이 운동 인증 중"
3. 그것도 미만 → 숫자 없이 **최근 인증자 아바타 + "운동 인증이 모이고 있어요"** (정성적)
4. 데이터 전무 → 라이브바 숨김 (기존 catch/empty 동작)

- 아바타·불꽃 라인 SVG 그대로. 비로그인 홈 최상단에 노출되도록 배치.
- 쿼리는 7일 윈도우까지만(최대 일주일치 feed 문서 읽기) — 누적 카운트용 별도 집계는 두지 않음(읽기 비용 억제, YAGNI).
- `LIVE_MIN`은 상단 상수로 두어 조정 가능.

### 2. 고의도 JIT 전환

#### 2a. 카피 톤 전환 (저위험, 텍스트만)
- `login-guard-modal` 헤드라인 "로그인이 필요해요"(벽 느낌) → 행동 맥락 초대형으로.
- 인증하기 진입 시 사유: 예) "첫 운동 인증, 같이 남겨요".
- 좋아요/댓글 등 기존 사유 문구는 톤만 점검(필요 시 소폭 수정), 구조 변경 없음.

#### 2b. 로그인 후 의도 재개 (핵심)
- `requireLogin(reason, onAfterLogin?)`에 선택적 콜백을 받아 보관(`window._pendingIntent`).
- 로그인 성공(onAuthStateChanged user 분기) 직후 보관된 콜백을 **1회** 실행 후 비운다.
- **적용 범위: "운동 인증하기" 1개에만.** 비로그인이 인증하기 탭 → 가드 모달 → 로그인 완료 시
  사진 선택기(`feed-photo-input`)를 자동 재오픈해 흐름을 잇는다.
- 좋아요·댓글 재개는 **v1 범위 제외**(로그인 후 같은 화면에 머무는 것으로 충분, YAGNI).

### 3. 비로그인 홈 배치 & 차단 지점 정리

- 비로그인 홈 노출 순서: 라이브바(활기) → 실시간 피드. 피드는 이미 로드됨.
- index.html:3509: 사진을 **고르게 뒀다가 막는 헛걸음** 제거 → "사진으로 운동 인증하기"
  트리거 시점(선택 전)에 비로그인이면 가드 모달(+의도 재개 콜백)을 띄운다.
- 에러/오프라인: 집계 실패 시 라이브바 조용히 숨김(기존 catch 유지).

## 범위 밖 (Non-goals)

- 비로그인 익명 게시(보안규칙 완화) — 하지 않음.
- 좋아요/댓글의 로그인 후 의도 재개 — v1 제외.
- 비로그인 전용 큐레이션 랜딩(B안) — 사용자 증가 후 별도 과제.
- 누적 전체 인증 수 집계/표시 — 읽기 비용 이유로 제외.

## 테스트 (수동 시나리오 중심)

단일 HTML + Firebase라 자동화가 약함. 다음을 수동 확인:

1. 라이브바 3단 폴백: (오늘 多) / (오늘 0·주 多) / (오늘·주 모두 미만) / (전무) 각 상태에서
   올바른 문구·숨김 동작.
2. 비로그인 → "운동 인증하기" 탭 → 가드 모달 → 로그인 → **사진 선택기 자동 재오픈** 확인.
3. 회귀: 관리자 / 일반 로그인 / 비로그인 각 상태에서 홈·피드·가드 모달 정상.
