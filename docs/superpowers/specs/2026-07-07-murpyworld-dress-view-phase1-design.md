# 머피월드 커스터마이징 UX 개편 — 1단계 설계 (꾸미기 전용 화면 + 방 캐릭터 키우기)

- 작성일: 2026-07-07
- 대상 파일: `index.html` (단일 HTML), `char/fields/*` (배경, 이번 단계는 **변경 없음**)
- 배포: main → GitHub Pages

## 배경 / 문제

폰(세로)에서 머피월드 캐릭터가 너무 작아 **커스터마이징이 잘 안 보인다**.
- 방(무대)이 `max-width:min(100%,42vh)` 정사각으로 눌려 있고, 그 안에서 캐릭터는 16타일 기준 3.4타일 높이(≈70px)로 렌더됨.
- 꾸미기 탭(`charHubTab('dress')`)에는 **별도 캐릭터 프리뷰가 없다**. `_charRenderAvatar`가 `#charworld-preview`를 찾지만 꾸미기 패널 마크업에 그 요소가 없어 실질적으로 미동작 → 옷을 갈아입어도 위쪽 작은 방의 걷는 캐릭터로만 확인해야 함.

## 큰 그림 (북극성) — 이번 단계 범위 아님

머피월드를 게임처럼: ① 첫 진입 "내 캐릭터 선택" 화면 → ② 방 등장(걷기) → ③ 방 안 **옷장 오브젝트** 앞에서 상호작용 → ④ **꾸미기 전용 UI**. 각 기능을 세계 속 오브젝트/별도 창으로 분리해 캐릭터를 크고 깔끔하게 보여준다.

**단계별로 진행하기로 확정:**
- **1단계 (이 문서):** ④ 꾸미기 전용 화면 + ② 방 캐릭터 키우기 (눈에 보이는 불편 즉시 해소)
- 2단계: ③ 옷장 오브젝트 상호작용으로 꾸미기 진입 (지금은 "꾸미기" 버튼이 트리거)
- 3단계: ① 캐릭터 선택 화면 첫 진입 연출

## 1단계 범위

### A. 꾸미기 전용 화면 (핵심)

"꾸미기"를 누르면 `page-char` 안에서 **꾸미기 모드로 전환**(방·드래그패드·허브탭을 덮음). "완료/뒤로"로 방으로 복귀.

세로 유지·위아래 분할:

```
┌──────────────────┐
│ ← 뒤로        💰 │  상단바 (뒤로/완료 + 머피 HUD)
├──────────────────┤
│      ╱▔▔▔╲       │
│     │ ◕  ◕ │     │  큰 캐릭터 (화면 위 ~55%)
│     │[상의] │     │  지금의 약 3~4배 크기
│      ╲___╱        │  옷 바꾸면 즉시 반영
│    ▁▁받침대▁▁      │
│    [앞][좌][뒤][우]│  ← 돌려보기 (4방향)
├──────────────────┤
│헤어 상의 하의 신발…│  슬롯 탭 (가로 스크롤)
│ ┌──┐┌──┐┌──┐     │
│ │it││it││it│     │  아이템 그리드 (아래 ~45%, 스크롤)
│ └──┘└──┘└──┘     │
│ [      저장      ]│
└──────────────────┘
```

**요소/동작:**
1. **큰 캐릭터 프리뷰** — 기존 레이어 합성 방식(`_charEquippedSheets` + 겹 div, `_charRenderAvatar`와 동일 원리)을 **큰 크기 + face(방향) 파라미터**로 렌더. 새 함수 `_charRenderHero(face)` 도입. 크기는 스테이지 영역(뷰포트 폭·높이 기반, 대략 화면 세로의 절반)에서 셀 비율(cw/ch) 유지.
2. **돌려보기 (돌리기)** — 앞(down)/좌(left)/뒤(up)/우(right) 4방향. 시트 행 오프셋으로 정지 프레임(col0) 표시. 컨트롤: 라벨 있는 좌/우 화살표 순환 **또는** 4버튼(앞·좌·뒤·우). 상태 `_charHeroFace` 유지, 기본 '앞(down)'.
3. **슬롯 탭 + 아이템 그리드** — 기존 `#charworld-slot-tabs` / `#charworld-items` 마크업과 `charSlot`/`charPick`/`charPickColor`/`charClearSlot` **그대로 재사용**. 위치만 꾸미기 화면 하단으로 이동.
4. **옷 변경 시 반영** — `charPick`/`charClearSlot`/`charPickColor`가 큰 프리뷰(`_charRenderHero`)와 (방이 살아있으면) 걷는 캐릭터 둘 다 갱신하도록 연결.
5. **저장/완료** — 저장은 기존 `charSave`. "뒤로/완료"는 꾸미기 모드 종료 → 방(월드 모드)·"내 캐릭터" 허브로 복귀.
6. **진입/이탈** — 진입은 현행 "꾸미기" 버튼(하단 허브탭). `charHubTab('dress')`가 하단 패널을 채우는 대신 **꾸미기 모드를 연다**(`mwOpenDress()`). `mwCloseDress()`로 종료. (2단계에서 옷장 상호작용이 `mwOpenDress`를 호출하도록 교체 예정 — 함수 경계를 그렇게 잡아 재사용.)
7. **고정 캐릭터(헬토리 등)** — 기존처럼 꾸미기 잠금 안내 유지(`body !== 'human'`).

### B. 방 캐릭터 크기 (B-1: 새 그림 없이 적당히 ~1.5배)

- 방 표시 크기: `.mw-stage` `max-width:min(100%,42vh)` → `min(100%,54vh)`.
- 캐릭터 키비율: `_charApplyPos`의 `DH = round(t * 3.4)` → `t * 4.0`.
- 타일 그리드(16) · 통행맵(16×16) · 배경(`field_home.png`) **변경 없음**. 캐릭터만 시각적으로 커짐(≈1.5배: 방 1.28× × 비율 1.18×).
- 본격 맵 재제작(B-2, 새 필드 아트 기반)은 **2단계/별도 단계로 미룸**.

## 지켜야 할 제약

- **근본 자산 불변:** `walk.png`/`heltori.png` 등 스프라이트는 표시 스케일만. 픽셀 그림 재제작·색 변경 금지.
- **픽셀 UI 킷 유지:** `.mw-*` CSS 킷, Galmuri 폰트, 라인SVG 배제·도트 톤. 이모지 금지. 색: 블루=메인/액션.
- **단일 파일:** 모든 변경 `index.html` 안. Firebase 모듈 스코프 규칙(window 브릿지) 준수.
- **캐시버스트:** 이번 단계는 새 이미지 없음 → `_ITEM_V` 불필요. 단 배포 시 `index.html` cache-bust 주석 갱신 + `sw.js` 버전 +1(가이드 규칙).
- **개발 플래그:** `MW_DEV_ALWAYS`는 이 단계에서 건드리지 않음(정식 배포 때 false).

## 검증

- 로컬 `python -m http.server` + 헤드리스 크롬 스크린샷으로:
  - 꾸미기 화면 4방향 돌려보기(앞/좌/뒤/우) 각각, 아이템 착용 상태에서 큰 프리뷰 정합 확인.
  - 좁은 폭(≈390px)과 큰 폭 뷰포트 둘 다에서 레이아웃 안 깨짐(가로 스크롤 없음, 캐릭터·그리드 겹침 없음).
  - 방(월드 모드) 새 크기에서 캐릭터가 커졌는지, 걷기/충돌 정상인지.
- "고쳤다" 보고 전 렌더로 눈 확인(가이드 규칙).

## 열린 사항 (1단계에서 결정 불필요, 기록만)

- 돌려보기 컨트롤 형태(화살표 순환 vs 4버튼)는 구현 중 보기 좋은 쪽으로. 기능은 동일(4방향 정지 프레임).
- 큰 프리뷰 배경/받침대 스타일은 간단한 픽셀 패널+옅은 그림자 수준.
