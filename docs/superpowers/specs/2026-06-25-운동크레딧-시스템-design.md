# 운동 크레딧 시스템 (Proof of Workout) — 설계

작성일: 2026-06-25
대상 파일: `index.html` (단일 HTML), `sw.js`(버전), Firestore `users` 컬렉션

## 목표

운동 인증을 머피의 **화폐이자 열쇠**로 만든다. 돈이 아니라 **"진짜 운동했다는 증거"** 로
매칭·대숲을 여는 선순환 루프를 만든다: 운동 인증 → 크레딧 → 매칭·대숲 열림 →
"더 하고 싶다" → 또 운동.

## 철학 / 원칙

- **막는 장치(벽)가 아니라 보상 장치.** "운동 안 하면 못 해"가 아니라 "운동하면 더 열려".
- 경쟁사(소개팅·소모임) 패턴 재조합이 아닌, **운동 커뮤니티에서만 의미 있는** 증명 기반 경제.
- 콜드스타트(초기 소수 사용자)에서 과하게 잠그지 않는다.
- 이모지 금지 → 머피 라인 SVG 아이콘만. 색상 블랙/화이트 + 블루(#3D7EFF)/골드(#F5C24B). 크레딧은 골드 포인트.

## 범위 (MVP)

크레딧이 여는 곳: **매칭("같이 운동해요" 신청) + 대숲(글쓰기).** 피드·크루는 기존대로 자유.

## 모드 (핵심 — "만들되 베타로 무제한")

`CREDITS_MODE` 단일 스위치로 제어한다. MVP는 코드 상수, 추후 Firestore 원격 설정으로 업그레이드(배포 없이 전환).

| 항목 | `beta` (현재) | `enforced` (추후, 사용자 충분히 모이면) |
|---|---|---|
| 크레딧 표시 | 보임 | 보임 |
| 적립(환영/일일) | 쌓임 | 쌓임 |
| 매칭·대숲 "1 크레딧" 라벨 | 보이되 **차감/차단 안 함** | **실제 차감·게이트** |
| "베타 · 무제한" 안내 | 표시 | 미표시 |

- **베타 보너스 효과:** 베타 동안 매일 인증한 씨앗 유저는 크레딧이 계속 쌓여, 정식 전환 시
  두둑한 잔고로 시작 → 충성 보상 + 급작스런 차단 방지.
- 기본값: `CREDITS_MODE = 'beta'`.

## 경제 (추천값 — 상수로 두어 조정 가능)

- **환영 보너스(신규 가입): +10** — 첫 경험에서 가장 기쁜 순간을 크게.
- **일일 적립: 하루 첫 운동 인증 = +1.** 그날 추가 인증은 피드엔 올라가도 크레딧 0(파밍 차단).
- **소모(정식 모드에서만 실제 차감):** 매칭 신청 = 1, 대숲 글 = 1.
- **첫 인증 축하 모먼트:** 하루 첫 인증으로 적립될 때 머피 라인 SVG + 골드 토스트로 "+1 크레딧" 축하.
- 상수 예시: `CREDIT_WELCOME = 10`, `CREDIT_DAILY = 1`, `COST_MATCH = 1`, `COST_BAMBOO = 1`.

## 적립 규칙 (하루 1회)

- 운동 인증(피드 글 작성 성공) 시, `creditsLastEarned`(마지막 적립 '날짜', 로컬 자정 기준)가
  오늘이 아니면 `credits += CREDIT_DAILY` 하고 `creditsLastEarned = 오늘`. 이미 오늘이면 적립 없음.
- 신규 유저: 사용자 문서 최초 생성 시 `credits = CREDIT_WELCOME`.

## 화면 (UI)

- **크레딧 카운터:** 골드 라인 SVG + 숫자. 마이페이지/매칭/대숲 등 관련 지점에 노출.
- **베타 배지:** `beta` 모드에서 "베타 · 무제한" 안내(이 시스템이 있고 지금은 공짜이며 추후 운동으로 번다는 인지 → 재방문 유도).
- **잔액 0(정식 모드):** 완전히 막지 않고 "오늘 운동 인증하면 바로 써요"로 인증 유도(핵심 행동으로 연결).
- 매칭·대숲 액션 버튼에 "1 크레딧" 라벨(베타엔 비차단).

## 데이터 구조 (Firestore)

- `users/{uid}` 문서에 필드 추가:
  - `credits`: number (잔액)
  - `creditsLastEarned`: string `YYYY-MM-DD` (하루 1회 적립 판정)
- 적립/소모는 `increment()` 또는 트랜잭션으로 안전 처리. (거래 로그는 MVP 생략, 잔액만)

## 어뷰징 방지 / 보안

- **하루 1회 적립**(`creditsLastEarned` 날짜 비교)으로 사진 파밍 차단.
- ⚠️ **정식(enforced) 전환 직전 필수:** 잔액을 사용자가 콘솔로 임의 증가시키지 못하도록
  **Firestore 보안규칙 또는 Cloud Functions로 잔액 무결성 보호**. 클라이언트만으론 불가.
  - 베타 동안은 무제한이라 위험 없음 → 이 보안 강화는 enforced 전환의 선행 과제로 명시.
- 사진 품질/실제 운동 여부 AI 검증은 범위 밖(추후).

## 콜드스타트

- 베타 모드 = 무제한이므로 초기 소수 사용자에도 잠금 없음. 자연스럽게 콜드스타트 흡수.

## 범위 밖 (Non-goals)

- 크레딧 구매/구독(파워유저 수익화) — **사업자 등록 후** 결제(PG)와 함께. (예창패 전략상 현재 사업자 전 단계라 결제 불가)
- AI 사진/운동 검증, 행동별 차등 비용, 거래 내역 화면.
- 크루 가입 등 매칭·대숲 외 행동 게이트.

## 영향받는 코드 지점 (구현 진입점)

- 인증(피드 글 작성 성공) 경로 — 일일 적립 훅. (`feed` 작성 성공 지점)
- 신규 유저 문서 생성 지점 — 환영 보너스. (`checkAndSetNickname`/카카오·구글 최초 생성)
- 매칭 신청 `handleLike`/`window.sendLike` — 비용 라벨 + (enforced 시) 차감·차단.
- 대숲 글쓰기 `postBamboo` — 비용 라벨 + (enforced 시) 차감·차단.
- 크레딧 표시 UI(마이페이지/매칭/대숲), 베타 배지.
- `CREDITS_MODE` 상수 + 경제 상수.
- cache-bust + sw.js 버전 갱신(자동 새로고침).

## 테스트 (수동 시나리오)

단일 HTML+Firebase라 자동화 약함. 수동 확인:
1. 신규 가입 → 잔액 = 환영 보너스(+10) 표시.
2. 하루 첫 인증 → +1 적립 + 축하 토스트. 같은 날 추가 인증 → 적립 없음.
3. 베타 모드: 매칭/대숲에 "1 크레딧" 라벨 보이되 무제한 사용, "베타 · 무제한" 표시.
4. (enforced 시뮬레이션) `CREDITS_MODE='enforced'`로 두고: 잔액 차감, 0이면 인증 유도 안내, 막힘.
5. 회귀: 비로그인/일반/관리자 각 상태에서 기존 매칭·대숲·피드 정상.
