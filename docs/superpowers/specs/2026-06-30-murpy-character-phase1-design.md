# 머피 캐릭터 — Phase 1 설계서

- 날짜: 2026-06-30
- 대상: Murpy (`C:\Users\won\Murpy\index.html`, 단일 HTML)
- 상태: 설계 승인 완료 → 구현 계획(writing-plans) 대기

## 1. 배경 / 목적

Murpy의 차별점은 **대숲 + 피드**다. 일반 소개팅앱과 다른, 운동 커뮤니티 정체성을
강화하고 머피(인앱 화폐)의 **소비처(sink)**를 만들기 위해 "싸이월드식 헬스 캐릭터"를 도입한다.

- 운동 → 머피 적립 → 캐릭터 꾸미기(소비) → 대숲에서 등장 → 남이 봄 = **동기 루프**
- 무과금러에겐 자기표현, 과금 동선엔 결제 포인트
- 본질(대숲·피드)을 **서포트**하는 역할. 잡아먹지 않는다.

## 2. 전체 로드맵 (3단계)

이 설계서는 **Phase 1만** 다룬다. Phase 2/3는 각자 별도 설계서를 가진다.

- **Phase 1 (본 문서)**: 캐릭터 만들기·꾸미기·머피 상점 + 대숲 등장(가면)
- **Phase 2**: 헬스장 뱃지·체크인 ("원정운동 도장깨기", 기존 센터 데이터 활용)
- **Phase 3**: 라이브 지도 (실시간 위치·남의 캐릭터·도트 모션·인플루언서)
  - **선결 원칙(미리 못박음)**: 위치 공개 기본 OFF(opt-in), 정확좌표 금지·"헬스장 단위"로만,
    차단/신고, 인플루언서 별도 동의. 안전설계 + 사용자 밀도 확보 전엔 착수 금지.
  - 오리지널 도트 에셋만 사용(포켓몬 "느낌"은 OK, 실제 에셋·이름·뱃지 복제는 법적 위험).

## 3. 핵심 결정 (확정)

| 항목 | 결정 |
|---|---|
| 캐릭터 형태 | 아바타형(부위 갈아입히기) |
| 대숲 연동 | **가면/변장 스킨** — 같은 캐릭터, 대숲에선 가면 씌워 정체 가림 |
| 그림체 | 도트(픽셀), 포켓몬 오마쥬 감성 (오리지널 에셋) |
| 렌더링 | 레이어드 도트 스프라이트(투명 PNG z순서 합성) |
| MVP 범위 | 작게 시작·확장형 |
| 화폐 단위 | 작은 정수 유지 ("운동 1회 = 1머피" 소울 보존), 유료 결제 설계는 후순위 |
| 캐릭터 공개 무대 | 대숲(가면)·마이페이지(풀)·트렌딩. **피드는 프사 유지** |

## 4. 범위 (Scope)

### 포함
- 1인 1캐릭터, 가입 시 기본 캐릭터 자동 지급
- 꾸미기 슬롯: **피부(베이스) · 헤어(모양+색) · 상의(모양+색) · 가면(대숲용)**
- 머피로 아이템 구매 → 보관함 소장 → 자유 착용
- 표시 위치: 마이페이지(풀), 대숲(가면), 트렌딩(인기 캐릭터 카드)
- 대숲 글/댓글에 작성자 자리에 **가면 쓴 캐릭터** 노출

### 제외 (YAGNI)
- 하의·신발·배경·액세서리 슬롯(슬롯 추가만으로 확장 가능하게 구조는 열어둠)
- 움직이는 모션/애니메이션
- 지도·실시간·뱃지·체크인 (Phase 2/3)
- 아이템 거래·선물·한정판·시즌
- 피드 아바타(프사 유지)

## 5. 데이터 모델

기존 `users/{uid}` 문서에 필드 추가. 새 컬렉션 없음.

```
users/{uid}
  character: { skin, hair, hairColor, top, topColor, mask }   // 현재 착용
  wardrobe:  [ itemId, ... ]                                   // 소장 아이템
  credits:   <number>                                          // 기존 머피(재사용)
```

- 신규 가입 시 `character`에 기본값, `wardrobe`에 기본 파츠 id 세팅.
- 각 슬롯 값은 `CHAR_ITEMS`의 itemId 또는 색상 코드.

### 아이템 카탈로그 (코드 상수)

`index.html` 내 상수 `CHAR_ITEMS` 배열. DB 시딩 불필요.

```js
// { id, slot:'skin'|'hair'|'top'|'mask', name, price, src, color? }
```

- `slot`: 어느 부위인지
- `price`: 머피 가격 (아래 §6)
- `src`: `/char/` 내 PNG 경로
- 기본(무료) 아이템은 `price:0`

## 6. 머피 경제

- **적립(유지)**: 가입 +10, 운동 인증 1회/일 +1
- **소비(기존, 유지)**: 매칭 신청/수락/'저에요' 1, 대숲 글쓰기 1
- **꾸미기 단가(신규)**:

| 아이템 | 머피 |
|---|---|
| 색상 변경 | 1 |
| 헤어 (일반) | 2 |
| 상의 (일반) | 2 |
| 가면 (대숲 특별) | 3 |
| (향후) 한정·프리미엄 | 5~10 |

- 구매 = 기존 `spendCredit(uid, price)` 재사용 → 성공 시 `wardrobe`에 추가. 착용은 무료.
- 현재 `CREDITS_MODE='beta'`(무제한) → 실제 차감 없음. 가격은 정해두고 `enforced` 전환 시 작동·튜닝.
- 적립처가 "하루 1머피" 하나뿐이라 꾸미기/매칭/대숲이 같은 지갑을 두고 경쟁한다.
  enforced 전환 시 적립처 추가(스트릭·뱃지 보너스 등) 또는 단가 미세조정으로 균형. (Phase 1 범위 밖, 메모로 남김)

## 7. 렌더링

- 고정 캔버스(예 96×96 도트), `position:relative` 컨테이너.
- 슬롯별 `position:absolute` `<img>` 레이어를 z순서로 합성: **베이스 < 헤어 < 상의 < 가면**.
- `image-rendering:pixelated`로 도트 선명.
- 색상: MVP는 단순화 — "색 옵션 = 색별 PNG" 또는 소수 색만. (코드 틴트 고도화는 후순위)
- 공용 함수:

```js
renderCharacter(config, { size, masked }) -> HTML 문자열
```

  - `config`: `users/{uid}.character`
  - `size`: 렌더 크기(px)
  - `masked:true`: 가면 강제 합성(대숲 전용). 가면 미보유 시 기본 가면 사용.
  - 마이페이지/대숲/트렌딩이 이 함수 하나를 공유.

## 8. UI / 진입 화면

- **마이페이지**: 내 캐릭터 풀버전 크게 + "꾸미기" 버튼 → 꾸미기 패널.
  - 꾸미기 패널: 슬롯 탭(피부/헤어/상의/가면) → 아이템 그리드 → 미리보기 + 착용/구매.
  - 보유 아이템은 착용, 미보유는 가격 표시 후 구매(머피 차감은 enforced에서).
- **대숲**: 글/댓글 작성자 자리에 `renderCharacter(config,{masked:true})`로 가면 캐릭터.
- **트렌딩("요즘 뜨는 콘텐츠")**: "인기 캐릭터" 카드. MVP는 최근/추천 featured(랜덤·최근 꾸밈), 진짜 인기순은 후순위.

## 9. 에셋 파이프라인

- 시작 세트: 베이스 1 · 헤어 4 · 상의 4 · 가면 3 (+색 일부).
- 진행 방식: **플레이스홀더 도트 파츠 먼저** 넣어 시스템 가동 → AI 도트 생성으로 초안 →
  투명배경·동일 그리드 정렬해 **PNG만 교체**. 그림 퀄리티가 출시를 막지 않게.
- 저장: 레포 `/char/` 폴더 PNG 번들 (GitHub Pages가 그대로 서빙).
- 일관성 규칙: 모든 파츠는 동일 캔버스/기준선/베이스 비율에 맞춤.

## 10. 미래 연결 훅 (지금은 자리만)

- `character` config는 Phase 2 뱃지·Phase 3 지도에서 그대로 재사용.
- `mask` 슬롯은 Phase 1에서 이미 대숲에 가동.

## 11. 성공 기준 (Phase 1)

- 신규/기존 사용자가 캐릭터를 갖고, 마이페이지에서 꾸밀 수 있다.
- 머피로 아이템을 구매(베타에선 무차감)하고 착용 상태가 저장·재로딩된다.
- 대숲에서 모든 사용자가 가면 캐릭터로 보인다(익명성 유지).
- 트렌딩에 캐릭터 카드가 노출된다.
- 기존 머피 적립/소비·대숲·피드·매칭 동작이 깨지지 않는다.
