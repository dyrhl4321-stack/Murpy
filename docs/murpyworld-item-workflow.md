# 머피월드 아이템 제작·검수 워크플로우 (확정 2026-07-09)

목적: **클로드 코드(클코)가 검증 없이 "됐어요"라고 앱에 올리는 걸 막는다.**
클코는 증거(검수 리포트·합성 미리보기)만 만들고, 판정·수정 스펙은 사람+코덱스가 한다.

## 방향
- 현재는 **하이브리드**: 기존 full sprite / 레거시 방식(423×896)으로 빠르게 안정화.
- v2 레이어 시스템(846×1792)은 **보관·앱 연결 보류**. 아이템 파이프라인 안정 후 재연결.

## 역할
- **김현수(사용자)**: 나노바나나로 에셋(시트+썸네일) 제작·수급. 최종 시각 승인. (필요 시) 나노바나나 재생성.
- **코덱스**: 기술 검수, 문제 진단, **클코가 실행할 수정 스펙(종합 프롬프트)** 작성. 표준·아키텍처 유지.
- **클로드 코드(클코)**: 검수 도구 실행 + 합성 미리보기 생성. 코덱스 스펙대로 **직접 픽셀 보정/추출**. 승인분만 앱 등록.

## 표준 (현행 레거시)
- 아이템 레이어 시트: **423×896**, 프레임 141×224, 3열(idle·walk1·walk2)×4행(down·up·left·right)
- 투명 배경, 아이템 픽셀만(몸/마네킹/체크무늬 금지), 프레임 이동·크롭 금지
- 썸네일: 별도 PNG(상점 아이콘)
- 렌더 베이스: `char/walk.png` (셀 141×224). 합성 검증은 이 위에 올림.

## 루프
```
1. [김현수] 시트 + 썸네일을 클코에게 전달
2. [클코] 검수 아티팩트 생성 (앱에 안 넣음)
     - validate: 크기·투명·빈PNG·프레임 정렬
     - char/walk.png 위 4방향 합성 미리보기 저장
     - 썸네일 확인
     → 리포트/이미지를 김현수·코덱스에게 제출. ★"괜찮다" 판단 안 함, 증거만.
3. [김현수 + 코덱스] 미리보기 보고 OK / 문제 판정
4. (문제) [코덱스] 수정 스펙 작성 → 클코에게
     예: "우향(row3) 프레임 x114-172 y268-342 흰 로고 제거" / "top 2px 아래" 등 정확히
5. [클코] 스펙대로 픽셀/추출 보정 → 미리보기 재생성 → 3번 복귀
6. (승인) [클코] 앱 등록: char/items 배치 + CHAR_ITEMS 연결 + 배포
```

## 불변 원칙
- 클코는 2~5단계를 **미리보기 이미지로만** 돈다. 말로 "됐어요/맞아요" 금지.
- 승인(6) 전엔 index.html·앱에 절대 안 넣는다.
- 레거시 423×896 아이템·walk.png 무손상. v2는 보관만.
- 검수 게이트 없이 등록 금지.

## 검수 도구
- `tools/character-customizer/customizer_cli.py` (validate / preview / compose). 레거시 검수엔 기본 매니페스트, v2는 `--manifest murpy_layers_v2.json`.
