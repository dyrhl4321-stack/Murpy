# 머피 캐릭터 4방향 도트 스프라이트 — 확보용 스펙

목적: 머피월드에서 캐릭터가 포켓몬처럼 4방향으로 걸어다니게 하는 **도트 스프라이트 시트** 확보.
현재는 코드로 그린 앞모습(SVG) 뿐 → 걷기/방향 표현엔 진짜 스프라이트가 필요.

---

## 1. 확보 파이프라인 (AI-first: 생성=AI → 정리=포토샵 → 연출=AE)
게임 바이브코딩의 표준 방식. 특히 **Nano Banana(Gemini 이미지)** 는 "같은 캐릭터 유지 + 다른 포즈/각도" 편집이 강해 4방향 스프라이트에 잘 맞음.
1. **AI 생성 (Nano Banana / GPT-image)**: 캐릭터 1종(레퍼런스)을 확정한 뒤, "이 캐릭터 그대로, 정면/뒤/옆·걷기 프레임"으로 **편집 생성**(참조 이미지 고정).
2. **포토샵 정리 = 핵심 수작업**: 48px 격자로 자르기·정렬·배경 투명·픽셀 선명화(안티에일리어싱 제거)·팔레트 통일 → **"일관된 시트"를 완성하는 단계.** AI는 여기까진 못 함.
3. **After Effects = 연출**: 입장 전환·반짝임·지도 이동 이펙트("무대 연출"). 스프라이트 자체가 아님.
- 보조 경로(막히면): SD+픽셀LoRA(대량 시도) / 오픈에셋 LPC 개조(CC 라이선스 표기) / 픽셀 아티스트 소량 외주.

## 1b. Nano Banana 워크플로우 (구체)
1. **레퍼런스 확정**: 정면 전신 대두 치비 1장 생성(투명·하드픽셀). 마음에 들 때까지 반복. 이게 "정답 캐릭터".
2. **각도 파생**: 레퍼런스를 첨부하고 "이 캐릭터 그대로, 뒤통수(위) 버전 / 옆모습(오른쪽) 버전" 요청 → 얼굴·비율 유지.
3. **걷기 프레임**: 각 방향마다 "왼발 앞 / 오른발 앞" 2포즈 추가 생성(정지 프레임 + 2 = 걷기 사이클).
4. **포토샵**: 전부 48px 격자에 정렬·투명·픽셀 정리 → 192×144 시트로 조립.
5. 옆모습은 오른쪽만 만들고 코드에서 좌우반전.

## 2. 먼저 정할 것: "걸어다니는 캐릭터가 꾸미기를 얼마나 반영하나" (아트량 결정)
- **L1 공용 1종(추천, 시작점)**: 오버월드 걷기 스프라이트는 **모두 공용 1종**. 꾸미기는 머피월드 무대·대숲 앞모습에서만 반영. → 아트 시트 1장이면 됨. "걷는 맛" 먼저 검증.
- **L2 색+헤어 소수 반영**: 베이스 걷기 + 헤어 몇 종 + 상의 **색 팔레트 스왑**. 아트 중간. "내 캐릭터가 걷는" 맛 up.
- **L3 풀 레이어**: 모든 아이템이 4방향 걷기까지 반영(LPC급). 아트 최대. 나중.
- **권장 진행: L1로 출시/검증 → 반응 좋으면 L2 → 성장하면 L3.**

## 3. 스프라이트 시트 규격 (아티스트/AI에게 주는 표준)
- **프레임 크기**: 48×48 px (대두 치비가 읽히는 최소 크기). 캐릭터는 프레임 안 약 36×44.
- **방향**: 아래(정면)·위(뒤통수)·옆 — 3방향. **옆은 좌우 반전으로 4방향 완성**(포켓몬 GBA 방식).
- **걷기 프레임**: 방향당 **4프레임**(정지 → 왼발 → 정지 → 오른발).
- **레이아웃**: 가로 = 프레임 4개, 세로 = 방향 3줄(아래/위/옆). 전체 **192×144 px**, 투명 PNG.
- **스타일**: 큰 머리 치비, 밝고 귀여움, **하드 픽셀(안티에일리어싱 금지)**, 제한 팔레트(8~16색), 얇은 외곽선 or 무외곽선 일관.
- **색 기준**: 피부 #F0C9A0 / 헤어·상의는 우리 팔레트(블루 #3D7EFF 등)와 맞추면 코드 색스왑 쉬움.
- 배경 반드시 **투명**, 프레임 간 캐릭터 위치·크기 **동일**(정렬).

## 4. AI 생성 프롬프트 (복붙용, 방향별로 각각 생성 후 조합)

**영문 (이미지 모델용):**
```
pixel art character sprite, cute chibi, big head, small body with legs,
top-down RPG overworld style (Pokemon GameBoy Advance vibe),
FACING DOWN (front), 4-frame walk cycle in a horizontal row,
transparent background, hard pixels no anti-aliasing, limited palette,
each frame 48x48, character centered and same size in every frame,
skin #F0C9A0, short dark-brown hair, blue t-shirt, friendly, clean outline
```
- 위 프롬프트에서 `FACING DOWN (front)` 를 `FACING UP (back of head, no face)`, `FACING RIGHT (side profile)` 로 바꿔 3장 생성.
- 각 장을 48px 격자로 잘라 시트로 합침(옆모습은 코드에서 좌우반전).

**한글 (아티스트 발주서용):**
```
귀여운 대두 치비 픽셀 캐릭터, 포켓몬 GBA풍 탑다운 오버월드.
프레임 48x48, 투명 PNG, 안티에일리어싱 없이 하드 픽셀, 제한 팔레트.
방향 3종(정면/뒤통수/옆) × 걷기 4프레임 = 시트 1장(192x144).
피부 #F0C9A0, 헤어·상의 색은 지정 팔레트. 모든 프레임 캐릭터 크기·위치 동일.
(옆은 좌우반전으로 재사용하므로 오른쪽 향만)
```

## 5. 코드 연결 방식 (아트 오면 내가 붙임)
- 시트 PNG를 `/char/walk_base.png` 로 저장.
- 걷기 = CSS `steps()` 배경 애니메이션 또는 canvas. `charMove`에서 방향에 따라 시트 행(row) 선택 + 걷기 프레임 재생, 이동 후 정지 프레임.
- 옆방향은 `transform:scaleX(-1)`로 좌우 재사용(현재 이동 코드에 이미 반전 있음).
- L2로 갈 땐 헤어/상의를 별도 레이어 시트로 두고 z합성(현재 renderCharacter 레이어 개념과 동일).
